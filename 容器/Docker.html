Docker 面试题及答案整理

1. Docker的基本概念是什么？

Docker是一个开源的容器化平台：

1、容器： Docker容器是轻量级的虚拟化技术，可以在其中运行应用程序。

2、镜像： Docker镜像是只读模板，用于创建容器。

3、仓库： Docker仓库存储和分发镜像，如Docker Hub。

4、特点： 轻量、快速、可移植、易于管理。

2. Docker和虚拟机的区别是什么？

Docker容器和虚拟机的主要区别：

1、架构： 虚拟机包含完整的操作系统，Docker共享宿主机内核。

2、启动速度： Docker秒级启动，虚拟机需要分钟级。

3、资源消耗： Docker更轻量，资源利用率更高。

4、隔离性： 虚拟机隔离级别更高，Docker隔离级别相对较弱。

3. Docker镜像的结构是怎样的？

Docker镜像是分层结构：

1、层叠： 多个镜像层叠加组成最终镜像。

2、只读： 镜像层是只读的，不可修改。

3、容器层： 容器运行时在镜像层上创建可写层。

4、共享： 多个镜像可以共享底层层，节省空间。

4. Docker的常用命令有哪些？

Docker常用命令包括：

1、镜像命令： docker pull、docker build、docker images、docker rmi。

2、容器命令： docker run、docker ps、docker start、docker stop、docker rm。

3、日志命令： docker logs、docker exec、docker inspect。

4、网络命令： docker network create、docker network ls。

5. Docker容器的生命周期是怎样的？

Docker容器生命周期：

1、创建： docker create 创建容器。

2、启动： docker start 启动容器。

3、运行： docker run 创建并启动容器。

4、暂停： docker pause 暂停容器。

5、停止： docker stop 停止容器。

6、删除： docker rm 删除容器。

6. Dockerfile的作用是什么？如何使用？

Dockerfile用于构建镜像：

1、指令： FROM、RUN、COPY、CMD、EXPOSE等。

2、构建： docker build -t name:tag .

3、多阶段构建： 使用多个FROM语句优化镜像大小。

4、最佳实践： 减少层数、使用.dockerignore。

7. Docker的数据持久化如何实现？

Docker数据持久化的方式：

1、卷（Volumes）： Docker管理的存储，持久化数据。

2、绑定挂载： 挂载宿主机目录到容器。

3、tmpfs挂载： 存储在内存中，不持久化。

4、使用场景： 数据库、文件存储等需要持久化的数据。

8. Docker网络模式有哪些？

Docker网络模式：

1、bridge： 默认模式，容器连接到docker0网桥。

2、host： 容器共享宿主机网络命名空间。

3、overlay： 跨主机网络，用于Docker Swarm。

4、none： 禁用网络功能。

5、container： 共享另一个容器的网络命名空间。

9. Docker Compose的作用是什么？

Docker Compose用于多容器应用：

1、定义： 使用YAML文件定义多容器应用。

2、命令： docker-compose up、down、ps、logs。

3、服务： 定义多个相互关联的服务。

4、网络： 自动创建共享网络。

10. Docker的优缺点是什么？

Docker的优点和缺点：

1、优点： 轻量、快速、可移植、环境一致性。

2、隔离： 应用程序间隔离性好。

3、缺点： 隔离性弱于虚拟机，存储驱动有性能开销。

4、安全： 共享内核存在安全风险。

11. Docker如何实现容器间通信？

Docker容器间通信方式：

1、bridge网络： 同一网络的容器通过IP互相访问。

2、容器命名： 可以通过容器名互相访问。

3、links： 使用--link建立连接（已废弃）。

4、自定义网络： 创建自定义bridge网络。

12. Docker镜像的构建优化有哪些方法？

Docker镜像构建优化方法：

1、多阶段构建： 分离构建和运行环境。

2、减少层数： 合并RUN指令。

3、使用较小基础镜像： 如alpine。

4、.dockerignore： 排除不需要的文件。

5、顺序优化： 将不常变化的指令放前面。

13. Docker的存储驱动有哪些？

Docker存储驱动类型：

1、overlay2： 推荐的存储驱动，性能好。

2、aufs： 早期使用的驱动，已废弃。

3、devicemapper： 早期CentOS使用的驱动。

4、btrfs、zfs： 文件系统级别的驱动。

14. Docker如何实现资源限制？

Docker资源限制方法：

1、内存限制： --memory、--memory-swap。

2、CPU限制： --cpus、--cpuset-cpus。

3、IO限制： --device-read-bps、--device-write-bps。

4、网络限制： docker network qos。

15. Docker的守护进程如何配置？

Docker守护进程配置：

1、配置文件： /etc/docker/daemon.json。

2、仓库配置： registry-mirrors加速镜像下载。

3、存储驱动： storage-driver配置存储驱动。

4、日志： log-driver、log-opts配置日志。

16. Docker如何实现日志管理？

Docker日志管理：

1、驱动： json-file、syslog、fluentd等。

2、查看： docker logs -f container。

3、限制： max-size、max-file限制日志大小。

4、集中管理： 配置日志收集系统。

17. Docker的安全最佳实践有哪些？

Docker安全最佳实践：

1、镜像安全： 使用官方镜像，定期扫描漏洞。

2、权限： 不要使用特权容器，避免root运行。

3、网络： 限制网络暴露，最小化端口映射。

4、资源： 设置资源限制，防止资源耗尽攻击。

18. Docker Registry的作用是什么？

Docker Registry用于存储和分发镜像：

1、官方仓库： Docker Hub是官方公共仓库。

2、私有仓库： 部署私有Registry存储内部镜像。

3、命令： docker pull、push操作镜像。

4、认证： 支持用户名密码认证。

19. Docker的COPY和ADD指令有什么区别？

COPY和ADD指令的区别：

1、COPY： 复制文件或目录到容器。

2、ADD： 复制并可解压tar文件，复制URL。

3、建议： 优先使用COPY，语义更清晰。

4、场景： 需要自动解压时使用ADD。

20. Docker的CMD和ENTRYPOINT有什么区别？

CMD和ENTRYPOINT的区别：

1、CMD： 提供容器默认命令，可被docker run参数覆盖。

2、ENTRYPOINT： 配置容器启动命令，不易被覆盖。

3、结合使用： ENTRYPOINT + CMD组合使用。

4、shell形式： CMD command param无法传递信号。

21. Docker如何实现健康检查？

Docker健康检查配置：

1、HEALTHCHECK： Dockerfile中定义健康检查。

2、命令： 检查容器是否健康的命令。

3、状态： docker ps显示容器健康状态。

4、自修复： 可配合restart策略实现自修复。

22. Docker Swarm和Kubernetes的区别是什么？

Docker Swarm和Kubernetes的区别：

1、学习曲线： Swarm更简单，K8S更复杂。

2、功能： K8S功能更全面，生态更丰富。

3、部署： Swarm与Docker集成更紧密。

4、选择： 根据需求和团队能力选择。

23. Docker如何实现多阶段构建？

Docker多阶段构建：

1、语法： 使用多个FROM语句。

2、阶段： 第一个阶段用于构建，第二个阶段用于运行。

3、COPY： 从上一阶段复制文件。

4、优势： 减小最终镜像大小。

24. Docker的USER指令如何使用？

Docker USER指令使用：

1、作用： 设置后续指令执行的用户。

2、格式： USER username:group或USER UID:GID。

3、安全： 以非root用户运行容器更安全。

4、权限： 确保用户有必要的文件权限。

25. Docker的EXPOSE指令有什么作用？

EXPOSE指令的作用：

1、声明： 声明容器运行时监听的端口。

2、文档： 相当于提供文档作用。

3、映射： -p参数会映射EXPOSE的端口。

4、协议： 可以指定TCP或UDP。

26. Docker的ONBUILD指令如何使用？

ONBUILD指令使用：

1、作用： 当镜像作为基础镜像时触发。

2、场景： 用于创建可复用的基础镜像。

3、语法： ONBUILD RUN/COPY等指令。

4、执行时机： 在子镜像构建时执行。

27. Docker如何实现私有仓库认证？

Docker私有仓库认证：

1、登录： docker login registry-url。

2、证书： 配置TLS证书。

3、用户： registry支持basic认证。

4、token： 支持token认证。

28. Docker的Volume生命周期如何管理？

Docker Volume生命周期管理：

1、创建： docker volume create。

2、挂载： -v或--mount挂载到容器。

3、查看： docker volume ls/inspect。

4、清理： docker volume prune删除未使用的卷。

29. Docker BuildKit是什么？有什么优势？

Docker BuildKit的特性：

1、并行构建： 支持并行构建多个层。

2、缓存： 更智能的构建缓存。

3、输出： 支持更多输出格式。

4、启用： DOCKER_BUILDKIT=1环境变量启用。

30. Docker的调试技巧有哪些？

Docker调试技巧：

1、logs： docker logs查看日志。

2、exec： docker exec -it进入容器。

3、inspect： docker inspect查看详细信息。

4、stats： docker stats查看资源使用。

31. Docker如何实现CI/CD集成？

Docker在CI/CD中的应用：

1、构建： 在CI流水线中构建Docker镜像。

2、测试： 使用容器运行测试。

3、推送： 推送镜像到仓库。

4、部署： 在CD阶段部署容器。

32. Docker的rootless模式是什么？

Docker rootless模式：

1、概念： 以非root用户运行Docker守护进程。

2、安全： 提高安全性，减少攻击面。

3、限制： 部分功能受限，如端口映射。

4、配置： 需要使用usermapconv插件。

33. Docker的清理命令有哪些？

Docker清理命令：

1、容器： docker container prune。

2、镜像： docker image prune。

3、卷： docker volume prune。

4、网络： docker network prune。

5、一键清理： docker system prune。

34. Docker和容器编排的关系是什么？

Docker与容器编排的关系：

1、Docker： 容器化技术，运行单个容器。

2、编排： 管理多个容器，协调它们的工作。

3、工具： Docker Swarm、K8S是编排工具。

4、场景： 微服务需要容器编排。

35. Docker的未来发展趋势是什么？

Docker的发展方向：

1、安全： 持续加强安全特性。

2、构建： BuildKit成为默认构建器。

3、Compose： Compose规范成为行业标准。

4、轻量： 镜像更小，启动更快。
