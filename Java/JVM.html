JVM 面试题及答案整理

1. JVM的整体结构是怎样的？

JVM主要组成部分：

1、类加载器： 负责加载class文件。

2、运行时数据区： 包括堆、栈、方法区等。

3、执行引擎： 包括解释器、JIT编译器、GC。

4、本地接口： JNI调用本地方法库。

2. JVM的内存区域划分？

JVM运行时数据区：

1、堆（Heap）： 对象实例和数组存储，GC主要区域。

2、虚拟机栈（VM Stack）： 方法调用栈帧，存储局部变量。

3、本地方法栈： 本地 native 方法调用。

4、方法区： 类信息、常量、静态变量，JDK8后改为元空间。

5、程序计数器： 当前线程执行的字节码行号指示器。

3. 堆和栈的区别？

堆和栈的主要区别：

1、存储内容： 堆存储对象实例，栈存储局部变量和方法调用。

2、线程共享： 堆线程共享，栈线程私有。

3、空间大小： 堆空间大，栈空间相对较小。

4、垃圾回收： 堆需要GC，栈自动管理。

4. 什么是类加载器？有哪些？

类加载器分类：

1、Bootstrap ClassLoader： 根加载器，加载JAVA_HOME/lib核心类库。

2、Extension ClassLoader： 扩展类加载器，加载JAVA_HOME/lib/ext。

3、App ClassLoader： 应用类加载器，加载classpath下类。

4、自定义类加载器： 用户自定义的类加载器。

5. 什么是双亲委派模型？

双亲委派模型原理：

1、原理： 类加载请求优先交给父加载器处理。

2、过程： 自底向上检查是否已加载，再自顶向下尝试加载。

3、优势： 避免类的重复加载，保证类的安全性。

4、破坏： SPI、代码热替换等场景会破坏双亲委派。

6. 什么是垃圾回收（GC）？

Java垃圾回收机制：

1、自动回收： JVM自动回收不再使用的对象内存。

2、回收区域： 主要回收堆内存，对方法区也会进行回收。

3、分代收集： 根据对象生命周期采用不同的回收策略。

4、回收算法： 包括标记-清除、复制、标记-整理等。

7. 如何判断对象是否需要回收？

对象存活判断算法：

1、引用计数法： 对象被引用时计数加1，计数为0可回收。无法处理循环引用。

2、可达性分析： 从GC Roots出发，标记不可达对象。可达性分析是主流算法。

3、GC Roots包括： 虚拟机栈引用的对象、方法区静态属性引用的对象等。

4、finalize： 对象被标记后可有最后一次逃脱机会。

8. Java的引用类型有哪些？

Java引用类型层次：

1、强引用： Object obj = new Object()，永不回收。

2、软引用（SoftReference）： 内存不足时回收，常用于缓存。

3、弱引用（WeakReference）： 下次GC时回收，常用于Map缓存。

4、虚引用（PhantomReference）： 形同虚设，回收时收到通知。

9. 常见的GC算法有哪些？

垃圾回收算法：

1、标记-清除： 标记存活对象，清除未标记对象。会产生内存碎片。

2、复制算法： 将内存分为两块，存活对象复制到另一块。适合年轻代。

3、标记-整理： 标记后移动存活对象整理空间。适合老年代。

4、分代收集： 根据对象年龄选择不同算法。

10. JVM的分代回收策略？

JVM分代回收策略：

1、年轻代： 包括Eden区和两个Survivor区，使用复制算法。

2、老年代： 存储长期存活对象，使用标记-整理算法。

3、永久代/元空间： 存储类信息，JDK8后改为元空间。

4、对象晋升： 年龄超过阈值后进入老年代。

11. 常见的垃圾收集器有哪些？

HotSpot垃圾收集器：

1、年轻代： Serial、ParNew、Parallel Scavenge。

2、老年代： Serial Old、Parallel Old、CMS。

3、整堆收集： G1、ZGC、Shenandoah。

4、默认： JDK8使用Parallel Scavenge + Parallel Old。

12. CMS收集器的工作原理？

CMS收集器过程：

1、初始标记： 标记GC Roots直接关联的对象，Stop The World。

2、并发标记： 标记全部存活对象，与用户线程并发执行。

3、重新标记： 修正并发标记期间变化的对象。

4、并发清除： 清除未标记的垃圾对象。

13. G1收集器是什么？

G1（Garbage First）收集器：

1、理念： 优先处理垃圾最多的区域。

2、化整为零： 将堆分为多个大小相等的Region。

3、回收方式： G1会维护一个优先列表，记录每个Region的垃圾量。

4、适用场景： 替代CMS，适合大内存多处理器环境。

14. 什么是Stop The World？

Stop The World（STW）：

1、概念： GC时暂停所有用户线程。

2、原因： 避免在标记时对象引用关系变化。

3、影响： STW会影响系统响应时间和吞吐量。

4、优化： GC算法一直在优化以减少STW时间。

15. 什么对象会进入老年代？

对象进入老年代的条件：

1、大对象： 对象大小超过阈值直接分配在老年代。

2、长期存活： 年龄超过MaxTenuringThreshold的对象。

3、动态年龄： Survivor中相同年龄对象总和大于Survivor空间一半。

4、Minor GC后： Survivor放不下时直接进入老年代。

16. Minor GC和Full GC的区别？

两种GC的区别：

1、Minor GC： 清理年轻代，空间不足时触发，较频繁。

2、Full GC： 清理整个堆和老年代，清理方法区。

3、触发条件： Minor GC对象年龄到阈值、老年代空间不足等。

4、停顿时间： Full GC停顿时间更长，影响更明显。

17. 什么是JIT编译器？

JIT（即时编译器）：

1、作用： 将热点字节码编译为本地机器码。

2、解释器： JVM默认解释执行字节码。

3、热点探测： 统计方法调用次数或循环回边次数。

4、编译优化： JIT会进行内联、死代码消除等优化。

18. Java内存泄漏是什么原因？

Java内存泄漏原因：

1、静态集合： 静态集合持有对象引用导致无法释放。

2、监听器： 注册监听器但未注销。

3、连接未关闭： 数据库连接、网络连接等未关闭。

4、变量作用域： 不必要的对象引用导致无法回收。

19. 如何排查JVM问题？

JVM问题排查工具：

1、命令行工具： jps、jstat、jmap、jstack、jinfo。

2、可视化工具： JConsole、VisualVM。

3、GC日志： 分析GC日志找出问题。

4、堆dump： jmap -dump分析内存快照。

20. JVM调优参数有哪些？

常见JVM调优参数：

1、堆大小： -Xms、-Xmx设置堆初始和最大大小。

2、年轻代： -Xmn设置年轻代大小，-XX:SurvivorRatio设置比例。

3、GC策略： -XX:+UseG1GC等选择垃圾收集器。

4、其他： -XX:MaxTenuringThreshold设置对象晋升年龄。

21. 什么是对象的内存布局？

对象在堆中的内存布局：

1、对象头： 包括Mark Word（哈希码、GC分代年龄等）和Klass Pointer（类型指针）。

2、实例数据： 对象实际字段数据。

3、对齐填充： 8字节对齐填充。

4、压缩指针： -XX:+UseCompressedOops压缩指针减少内存占用。

22. 什么是TLAB？

TLAB（Thread Local Allocation Buffer）：

1、概念： 为每个线程在堆中分配的私有缓冲区。

2、作用： 减少对象分配时的竞争，提高并发分配效率。

3、开启： -XX:+UseTLAB开启，默认开启。

4、大小： -XX:TLABSize设置大小，-XX:ResizeTLAB动态调整。

23. ZGC和G1的区别？

ZGC和G1对比：

1、停顿时间： ZGC停顿时间在毫秒级，G1停顿时间较长。

2、分代： G1是分代收集器，ZGC不分代。

3、内存： ZGC支持更大内存（可达16TB），G1适合小内存。

4、原理： ZGC使用着色指针和读屏障实现并发标记。

24. 什么是栈上分配？

栈上分配（Stack Allocation）：

1、概念： 对象在栈上分配而非堆上分配。

2、条件： 逃逸分析确定对象不会逃逸出方法。

3、优势： 无需GC，方法退出时自动释放。

4、标量替换： 进一步优化为标量（基本类型）替代对象。

25. 什么是逃逸分析？

逃逸分析（Escape Analysis）：

1、概念： 分析对象的作用域是否超出方法或线程。

2、全局逃逸： 对象作为返回值或静态变量引用。

3、参数逃逸： 对象作为参数传递给其他方法。

4、优化： 基于逃逸分析进行锁消除、栈上分配等优化。

26. 什么是Card Table？

Card Table（卡片表）：

1、概念： 老年代中用于记录对象引用的结构。

2、原理： 将老年代划分为固定大小的卡片，标记被年轻代引用的卡片。

3、作用： 加速Minor GC，不需要扫描整个老年代。

4、设置： -XX:+UseCardTable开启，默认启用。

27. 什么情况下会触发Full GC？

触发Full GC的条件：

1、老年代空间不足： 对象分配时老年代空间不足。

2、方法区空间不足： JDK7及以前永久代空间不足。

3、Minor GC前： 老年代最大可用空间小于历次晋升平均值。

4、显式调用： System.gc()建议执行Full GC。

28. 对象创建过程是怎样的？

对象创建过程：

1、检查类加载： 检查类是否已加载，未加载则先加载。

2、分配内存： 在堆中分配内存，指针碰撞或空闲列表方式。

3、初始化零值： 将对象字段初始化为零值。

4、设置对象头： 设置Mark Word和类型指针。

5、执行init： 执行构造函数和字节码。

29. 为什么堆内存要分代？

分代收集的原因：

1、对象生命周期： 大多数对象创建后很快就会变成垃圾。

2、扫描效率： 分代后可以只扫描年轻代，减少扫描范围。

3、算法优化： 不同代适用不同垃圾收集算法。

4、性能提升： 减少GC停顿时间，提高吞吐量。

30. 什么是内存分配担保？

内存分配担保机制：

1、原因： Minor GC时Survivor空间不足，对象需进入老年代。

2、机制： Minor GC前检查老年代最大可用空间。

3、大于历次晋升： 老年代空间大于历次晋升平均值，直接Minor GC。

4、小于历次晋升： 检查HandlePromotionFailure参数，可能触发Full GC。

31. 什么是Card Marking？

Card Marking机制：

1、问题： Minor GC时需要扫描老年代对年轻代对象的引用。

2、解决方案： 使用Card Table记录老年代对象的修改。

3、Dirty Card： 老年代对象引用年轻代时标记为脏卡。

4、扫描范围： 只需扫描脏卡对应的内存区域。

32. 什么是Prometheus GC？

Prometheus GC（晋升到老年代的GC）：

1、概念： 对象从年轻代移动到老年代的过程。

2、年龄阈值： 对象年龄达到MaxTenuringThreshold后晋升。

3、动态年龄： Survivor中相同年龄对象占比超过一半则提前晋升。

4、大对象： 大对象直接分配在老年代。

33. ZGC的着色指针是什么？

ZGC着色指针原理：

1、概念： 在对象指针上使用标记位存储GC状态。

2、42位地址： ZGC使用42位地址，剩余位用于标记。

3、Marked0/Marked1： 两个标记位用于并发标记。

4、Remapped： 标记对象是否已移动到新位置。

34. 什么是读屏障？

读屏障（Load Barrier）：

1、概念： 在读取对象引用时执行的代码。

2、作用： ZGC中用于保证读取到最新对象位置。

3、自愈：

读取到旧引用时自动修正为新引用。

4、开销： 读屏障开销较小，约2-3%性能影响。

35. JVM调优的一般步骤？

JVM调优基本流程：

1、确定目标： 明确延迟时间和吞吐量目标。

2、监控分析： 通过工具监控GC情况和内存使用。

3、问题定位： 分析GC日志找出问题原因。

4、参数调整： 调整堆大小、GC收集器等参数。

5、验证优化： 验证调优效果，持续迭代。
