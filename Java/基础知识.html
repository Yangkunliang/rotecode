Java基础知识 面试题及答案整理

1. Java的基本数据类型有哪些？

Java的8种基本数据类型：

1、byte： 字节型，占1个字节，范围-128到127。

2、short： 短整型，占2个字节，范围-32768到32767。

3、int： 整型，占4个字节，范围约-21亿到21亿。

4、long： 长整型，占8个字节。

5、float： 单精度浮点型，占4个字节。

6、double： 双精度浮点型，占8个字节。

7、char： 字符型，占2个字节。

8、boolean： 布尔型，true或false。

2. String、StringBuilder、StringBuffer的区别？

三者区别：

1、String： 不可变字符串，每次修改都会创建新对象。

2、StringBuilder： 可变字符串，非线程安全，性能较好。

3、StringBuffer： 可变字符串，线程安全，使用synchronized。

4、使用建议： 多线程共享字符串用StringBuffer，单线程用StringBuilder。

3. ==和equals的区别？

==和equals的区别：

1、==： 比较的是引用地址，基础类型比较值。

2、equals： 默认比较引用地址，可重写比较内容。

3、String重写： String类重写了equals方法比较字符串内容。

4、注意： equals需要和hashCode配合使用时需重写两者。

4. Java的访问修饰符有哪些？

Java访问修饰符：

1、private： 私有访问权限，仅同类可见。

2、default： 默认访问权限，同包可见。

3、protected： 受保护访问权限，同包及子类可见。

4、public： 公共访问权限，所有地方可见。

5. Java的三大特性是什么？

面向对象三大特性：

1、封装： 把数据和操作数据的方法封装在类中。

2、继承： 子类继承父类的属性和方法。

3、多态： 同一个行为具有不同表现形式。包括重载和重写。

6. 重载和重写的区别？

重载和重写的区别：

1、重载（Overload）： 同一个类中，方法名相同但参数不同。

2、重写（Override）： 子类重新实现父类的方法。

3、编译时多态： 重载是编译时多态。

4、运行时多态： 重写是运行时多态。

7. 接口和抽象类的区别？

接口和抽象类的区别：

1、继承： 类只能单继承抽象类，但可以实现多个接口。

2、方法： 抽象类可以有具体方法，接口JDK7前只能是抽象方法。

3、成员变量： 抽象类可以有各种类型成员，接口只能是public static final。

4、使用场景： 抽象类用于is-a关系，接口用于can-do关系。

8. Java异常处理机制？

Java异常处理：

1、try： 尝试执行可能异常的代码。

2、catch： 捕获并处理异常。

3、finally： 无论是否异常都会执行的代码。

4、throw： 抛出异常。

5、throws： 声明方法可能抛出的异常。

9. CheckedException和RuntimeException的区别？

两种异常类型的区别：

1、CheckedException： 编译时异常，必须处理或声明抛出。

2、RuntimeException： 运行时异常，可处理也可不处理。

3、常见例子： IOException是Checked，NullPointerException是Runtime。

4、建议： 积极处理CheckedException，RuntimeException通常表示程序错误。

10. Java的集合框架有哪些？

Java集合框架核心接口：

1、Collection： 单列集合根接口。

2、List： 有序可重复集合，ArrayList、LinkedList、Vector。

3、Set： 无序不可重复集合，HashSet、TreeSet、LinkedHashSet。

4、Map： 双列集合键值对，HashMap、TreeMap、LinkedHashMap。

5、Queue： 队列接口，PriorityQueue、LinkedList。

11. ArrayList和LinkedList的区别？

ArrayList和LinkedList对比：

1、数据结构： ArrayList使用数组，LinkedList使用双向链表。

2、随机访问： ArrayList按索引访问快，LinkedList需要遍历。

3、插入删除： LinkedList插入删除快，ArrayList可能需要移动元素。

4、内存： ArrayList内存连续，LinkedList需要额外存储指针。

12. HashMap的底层数据结构？

HashMap底层实现：

1、JDK1.7： 数组+链表（Entry数组）。

2、JDK1.8： 数组+链表+红黑树（Node数组+TreeNode）。

3、扰动函数： hash()方法使用高低位异或减少碰撞。

4、树化条件： 单链超过8且数组长度大于64时转为红黑树。

13. HashMap的put过程？

HashMap的put执行流程：

1、计算hash： 调用hash()计算key的hash值。

2、定位索引： 根据hash计算数组索引位置。

3、遍历链表/红黑树： 遍历该位置的链表或红黑树。

4、插入或更新： 找到key则更新value，否则插入新节点。

5、扩容判断： 插入后判断是否需要扩容。

14. ConcurrentHashMap如何保证线程安全？

ConcurrentHashMap线程安全机制：

1、JDK1.7： Segment数组+HashEntry，每个Segment独立加锁。

2、JDK1.8： 使用CAS+synchronized对每个数组节点加锁。

3、CAS： 使用CAS操作进行无锁并发。

4、synchronized： 对头节点加锁，保证原子性操作。

15. HashMap和Hashtable的区别？

HashMap和Hashtable对比：

1、线程安全： Hashtable线程安全，HashMap非线程安全。

2、null支持： HashMap允许key和value为null，Hashtable不允许。

3、性能： HashMap性能更好。

4、迭代器： HashMap使用fail-fast迭代器，Hashtable使用Enumerator。

16. Java的IO流有哪些？

Java IO流分类：

1、输入流/输出流： 按方向分为输入流和输出流。

2、字节流/字符流： 按数据类型分为字节流和字符流。

3、节点流/处理流： 节点流直接操作数据源，处理流包装其他流。

4、缓冲流： BufferedInputStream、BufferedReader等提供缓冲功能。

17. BIO、NIO、AIO的区别？

三种IO模型对比：

1、BIO： 同步阻塞IO，线程等待IO完成。

2、NIO： 同步非阻塞IO，使用Selector多路复用。

3、AIO： 异步非阻塞IO，基于事件和回调。

4、适用场景： BIO适合连接数少，NIO适合高并发，AIO适合长连接。

18. Java的反射机制？

Java反射机制：

1、概念： 在运行时动态获取类信息和调用方法。

2、Class类： 反射的核心，获取类信息。

3、获取方式： Object.getClass()、Class.forName()、.class。

4、用途： 框架、注解、动态代理都用到反射。

19. Java的泛型有什么用？

Java泛型的作用：

1、类型安全： 编译时检查类型，避免强制类型转换。

2、代码复用： 编写通用代码，减少重复代码。

3、泛型类： 类名后加，如ArrayList。

4、泛型方法： 方法返回值前加。

20. Java注解的作用？

Java注解的用途：

1、编译检查： 如@Override检查是否重写。

2、代码生成： 如Lombok自动生成getter/setter。

3、运行时处理： 如Spring注解进行依赖注入。

4、自定义注解： @interface定义，配合反射使用。

21. Java的序列化是什么？

Java序列化机制：

1、概念： 将对象转换为字节流，便于存储和传输。

2、实现： 类实现Serializable接口。

3、serialVersionUID： 版本号，确保序列化兼容性。

4、transient： 标记的字段不参与序列化。

22. Java值传递还是引用传递？

Java参数传递规则：

1、结论： Java只有值传递。

2、基本类型： 传递的是值的副本。

3、引用类型： 传递的是引用地址的副本。

4、误解： 不是传递引用本身，而是引用的副本。

23. 静态变量和实例变量的区别？

两种变量类型的区别：

1、static： 静态变量属于类，所有对象共享。

2、实例变量： 实例变量属于对象，每个对象独立。

3、存储位置： 静态变量在方法区，实例变量在堆中。

4、访问方式： 静态变量可通过类名访问，实例变量需通过对象访问。

24. Java中final关键字的作用？

final关键字用法：

1、修饰变量： 变量值不可改变，成为常量。

2、修饰方法： 方法不可被重写。

3、修饰类： 类不可被继承。

4、修饰参数： 参数在方法内不可被修改。

25. Java的内存区域划分？

JVM内存区域：

1、堆（Heap）： 存储对象实例和数组，垃圾回收的主要区域。

2、栈（Stack）： 存储局部变量、方法调用栈帧。

3、方法区（Method Area）： 存储类信息、常量、静态变量。

4、程序计数器： 记录当前线程执行的字节码行号。

5、本地方法栈： 存储本地方法调用信息。

26. Java的类加载机制？

类加载过程：

1、加载： 将class文件加载到内存。

2、验证： 验证字节码的正确性。

3、准备： 为静态变量分配内存并初始化。

4、解析： 将符号引用替换为直接引用。

5、初始化： 执行静态代码块和静态变量赋值。

27. Java的类加载器有哪些？

三层类加载器：

1、Bootstrap ClassLoader： 加载JAVA_HOME/lib下核心类库。

2、Extension ClassLoader： 加载JAVA_HOME/lib/ext下扩展类。

3、App ClassLoader： 加载classpath下应用程序类。

4、双亲委派： 类加载器优先委托给父加载器加载。

28. 为什么HashMap数组长度是2的幂次？

HashMap数组长度设计原因：

1、hash计算： index = (n-1) & hash，2的幂次时效率最高。

2、位运算： (n-1)的二进制都是1，&运算分布均匀。

3、扩容优化： 扩容时只需判断hash某一位是0还是1。

4、性能： 减少hash碰撞，提高查询效率。

29. Java创建对象的方式有哪些？

Java创建对象的几种方式：

1、new： 最常用的创建方式。

2、反射： Class.newInstance()或Constructor.newInstance()。

3、clone： 实现Cloneable接口后调用clone()。

4、反序列化： 从字节流恢复对象。

30. Java8有哪些新特性？

Java 8主要新特性：

1、Lambda表达式： 简洁的函数式编程语法。

2、Stream API： 流式数据处理。

3、Optional： 解决空指针问题。

4、接口默认方法： 接口可以有默认实现。

5、新的日期API： LocalDate、LocalDateTime等。

31. Java中volatile的作用？

volatile关键字作用：

1、可见性： 保证变量对所有线程可见。

2、禁止重排序： 防止CPU对指令重排序。

3、不能保证原子性： volatile不能替代synchronized。

4、使用场景： 状态标记量、单例模式中的double check。

32. Java的Object类有哪些方法？

Object类的核心方法：

1、equals()： 判断对象是否相等。

2、hashCode()： 返回对象哈希值。

3、toString()： 返回对象字符串表示。

4、clone()： 克隆对象。

5、getClass()： 获取类对象。

6、notify/notifyAll/wait： 多线程相关。

33. 为什么HashMap线程不安全？

HashMap线程不安全原因：

1、数据覆盖： 并发put可能导致数据覆盖。

2、死循环： JDK1.7并发resize可能形成环形链表，导致死循环。

3、数据丢失： 并发插入可能导致数据丢失。

4、替代方案： 使用ConcurrentHashMap代替。

34. Java中强引用、软引用、弱引用、虚引用？

四种引用类型：

1、强引用： Object obj = new Object()，永不回收。

2、软引用： 内存不足时回收，用于缓存。

3、弱引用： 下次垃圾回收时回收，用于Map缓存。

4、虚引用： 形同虚设，回收时收到通知。

35. Java的自动拆箱装箱是什么？

自动拆箱和装箱：

1、装箱： 基本类型自动转为包装类型，如int转Integer。

2、拆箱： 包装类型自动转为基本类型，如Integer转int。

3、缓存： Integer有缓存-128到127的对象。

4、注意： 装箱过程可能创建新对象，影响性能。
