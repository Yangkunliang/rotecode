并发编程 面试题及答案整理

1. 什么是进程和线程？

进程和线程的区别：

1、进程： 资源分配的基本单位，拥有独立的地址空间。

2、线程： CPU调度的基本单位，是进程内的执行路径。

3、资源： 进程间资源独立，线程共享进程资源。

4、开销： 线程创建销毁开销远小于进程。

2. 为什么要使用多线程？

多线程的优势：

1、提高CPU利用率： 充分利用多核CPU资源。

2、提高响应速度： 后台处理不影响主线程响应。

3、简化编程模型： 将复杂任务分解为独立线程。

4、资源复用： 线程可共享进程资源。

3. 线程的状态有哪些？

线程的生命周期状态：

1、NEW： 初始状态，创建但未启动。

2、RUNNABLE： 可运行状态，可能正在运行或等待CPU。

3、BLOCKED： 阻塞状态，等待获取锁。

4、WAITING： 无限期等待，等待其他线程通知。

5、TIMED_WAITING： 有限期等待，一定时间后自动返回。

6、TERMINATED： 终止状态，线程执行完毕。

4. sleep()和wait()的区别？

sleep和wait的区别：

1、所属类： sleep是Thread的方法，wait是Object的方法。

2、释放锁： sleep不释放锁，wait释放锁。

3、使用场景： sleep用于线程暂停，wait用于线程间通信。

4、恢复方式： sleep超时自动恢复，wait需要notify唤醒。

5. synchronized关键字的作用？

synchronized同步关键字：

1、修饰实例方法： 锁定当前对象，同一实例方法互斥。

2、修饰静态方法： 锁定当前类对象，整个类的实例互斥。

3、修饰代码块： 锁定指定对象，只对代码块内代码同步。

4、可重入： synchronized是可重入锁，同一线程可多次获取。

6. synchronized和Lock的区别？

synchronized和Lock的区别：

1、获取方式： synchronized自动获取释放锁，Lock需手动lock/unlock。

2、等待可中断： Lock可中断等待，synchronized不行。

3、锁类型： Lock支持公平锁和非公平锁。

4、条件变量： Lock有Condition，可分组唤醒线程。

7. volatile关键字的作用？

volatile的作用：

1、可见性： 保证变量对所有线程可见，修改立即写回主存。

2、禁止重排序： 防止CPU指令重排序，保证有序性。

3、不能保证原子性： volatile不能替代synchronized。

4、适用场景： 状态标记量、单例模式double check。

8. 什么是线程安全？

线程安全的概念：

1、概念： 多个线程访问同一对象时，结果始终正确。

2、不可变： 不可变对象一定是线程安全的。

3、绝对线程安全： 任何情况下都线程安全。

4、相对线程安全： 大部分方法需要同步才能线程安全。

9. 什么是死锁？

死锁的概念和条件：

1、概念： 多个线程相互等待对方持有的资源而无法推进。

2、互斥条件： 资源一次只能被一个线程持有。

3、持有并等待： 线程持有资源同时等待其他资源。

4、不可抢占： 已分配资源不能被强制释放。

5、循环等待： 形成资源-线程循环链。

10. 如何避免死锁？

死锁的预防措施：

1、破坏持有并等待： 一次申请所有资源。

2、破坏不可抢占： 已获取部分资源时可释放。

3、破坏循环等待： 按固定顺序获取资源。

4、设置超时： 使用Lock.tryLock()设置超时。

11. ThreadLocal是什么？

ThreadLocal线程本地变量：

1、概念： 为每个线程提供独立的变量副本。

2、实现： 每个线程有ThreadLocalMap存储变量。

3、用途： 解决数据库连接、Session等线程安全问题。

4、注意： 使用完后要调用remove()防止内存泄漏。

12. 什么是CAS？

CAS（Compare-And-Swap）原理：

1、概念： 乐观锁实现，无锁并发编程。

2、原理： 三个值：内存位置V、预期值A、新值B，V=A时才会更新。

3、原子性： CPU提供硬件级别原子指令。

4、问题： 存在ABA问题、循环开销大、只能保证一个变量。

13. ReentrantLock的实现原理？

ReentrantLock可重入锁原理：

1、继承： 实现了Lock接口，继承AQS。

2、计数器： 内部使用state计数器记录重入次数。

3、重入实现： 获取锁时state+1，释放时state-1。

4、公平锁： 公平版按等待顺序获取锁。

14. 什么是AQS？

AbstractQueuedSynchronizer（AQS）：

1、概念： Java并发包的核心基础框架。

2、核心思想： 基于FIFO双向队列和int类型state实现同步。

3、模板方法： 提供acquire、release等模板方法。

4、子类实现： 子类实现tryAcquire、tryRelease等方法。

15. 什么是Condition？

Condition条件变量：

1、概念： 配合Lock使用，实现等待/通知模式。

2、await： 当前线程等待，同时释放锁。

3、signal： 唤醒一个等待线程。

4、signalAll： 唤醒所有等待线程。

16. 什么是阻塞队列？

阻塞队列（BlockingQueue）：

1、概念： 支持阻塞操作的队列。

2、offer： 队列满时阻塞等待。

3、poll： 队列空时阻塞等待。

4、实现： ArrayBlockingQueue、LinkedBlockingQueue等。

17. 什么是线程池？

线程池的作用和好处：

1、复用： 复用线程，减少线程创建销毁开销。

2、管理： 控制并发数量，避免过多线程耗尽资源。

3、参数： corePoolSize、maximumPoolSize、keepAliveTime等。

4、拒绝策略： 队列满时的处理策略。

18. 线程池的工作流程？

线程池处理任务流程：

1、判断核心线程： 核心线程未满则创建新线程执行任务。

2、判断队列： 核心线程满则加入等待队列。

3、判断最大线程： 队列满且未达最大线程则创建新线程。

4、执行拒绝策略： 达到最大线程数且队列满则执行拒绝策略。

19. 线程池的类型有哪些？

常用线程池类型：

1、FixedThreadPool： 固定大小的线程池。

2、CachedThreadPool： 可缓存的线程池，线程数量动态增长。

3、SingleThreadExecutor： 单线程的线程池。

4、ScheduledThreadPool： 定时调度的线程池。

5、ForkJoinPool： 分治思想的线程池，适合计算密集型。

20. execute()和submit()的区别？

execute和submit的区别：

1、返回值： execute无返回值，submit返回Future。

2、异常处理： execute异常直接抛出，submit可捕获Future。

3、用途： execute用于执行Runnable，submit两者都行。

4、推荐： 通常推荐使用submit()。

21. 什么是CountDownLatch？

CountDownLatch倒计时门闩：

1、概念： 一种同步辅助，允许一个或多个线程等待。

2、countDown： 计数减1操作。

3、await： 等待计数归零。

4、特点： 计数不能重置，一次性使用。

22. 什么是CyclicBarrier？

CyclicBarrier循环栅栏：

1、概念： 等待所有线程到达后再一起继续执行。

2、await： 线程到达时等待。

3、重置： 计数器归零后自动重置，可循环使用。

4、应用： 多线程计算最后汇总结果。

23. 什么是Semaphore？

Semaphore信号量：

1、概念： 控制同时访问特定资源的线程数量。

2、acquire： 获取一个许可。

3、release： 释放一个许可。

4、应用： 限流、连接池等场景。

24. 什么是Exchanger？

Exchanger交换器：

1、概念： 两个线程间交换数据的同步工具。

2、exchange： 线程到达时交换数据。

3、超时： 可设置超时时间。

4、应用： 遗传算法、数据结构填充等。

25. 什么是Phaser？

Phaser多阶段栅栏：

1、概念： Java7引入，支持多阶段并发控制。

2、动态注册： 支持动态添加和移除参与者。

3、arriveAndAwaitAdvance： 到达并等待其他参与者。

4、应用： 复杂的多阶段任务。

26. 如何保证线程安全？

线程安全保证方法：

1、synchronized： 同步方法或代码块。

2、Lock： 显示锁实现同步。

3、volatile： 保证可见性和有序性。

4、原子类： AtomicInteger等原子类保证原子操作。

5、不可变对象： 使用不可变对象避免并发问题。

27. ConcurrentHashMap如何保证线程安全？

ConcurrentHashMap线程安全机制：

1JDK1.7： Segment数组，每个Segment类似HashMap加锁。

2JDK1.8： 使用CAS+synchronized对每个数组节点加锁。

3、CAS： 无锁并发更新数组元素。

4、synchronized： 对头节点加锁，保证原子性。

28. CopyOnWriteArrayList是什么？

CopyOnWriteArrayList特点：

1、写入复制： 写入时复制整个数组。

2、读操作： 读操作不需要加锁。

3、适用场景： 读多写少的并发场景。

4、缺点： 写操作开销大，不适合写多场景。

29. 什么是伪共享？如何避免？

伪共享问题：

1、概念： 同一缓存行的多个变量被不同线程修改。

2、影响： 导致缓存行失效，降低性能。

3、避免方法： 使用填充使变量独占缓存行。

4、@Contended： JDK8提供注解自动填充。

30. Java内存模型是什么？

Java Memory Model（JMM）：

1、概念： 定义了线程和主内存之间的抽象关系。

2、主内存： 所有变量存储在主内存。

3、工作内存： 每个线程独有的工作内存。

4、交互： 线程操作变量前从主内存复制到工作内存。

31. happens-before原则是什么？

happens-before规则：

1、概念： 保证前一个操作对后一个操作可见。

2、程序顺序： 单线程中前一个操作happens-before后一个。

3、监视器锁： 解锁happens-before加锁。

4、volatile： 写happens-before读。

32. 什么是线程中断？

线程中断机制：

1、interrupt： 中断目标线程，设置中断标志。

2、isInterrupted： 判断是否被中断，不清除标志。

3、interrupted： 判断并清除中断标志。

4、响应： 线程需主动检查中断状态并处理。

33. 如何停止一个线程？

线程停止的正确方式：

1、volatile标志： 使用volatile boolean标志控制循环。

2、interrupt： 使用中断标志配合isInterrupted检查。

3、return： 配合return停止线程。

4、不推荐： stop、suspend、resume已废弃。

34. 什么是守护线程？

守护线程（Daemon Thread）：

1、概念： 为用户线程提供服务的线程。

2、setDaemon： 将线程设置为守护线程。

3、特点： JVM中所有用户线程结束，守护线程自动终止。

4、示例： 垃圾回收线程是典型的守护线程。

35. 并发编程最佳实践？

并发编程建议：

1、最小化同步范围： 尽量缩小synchronized代码块。

2、优先使用volatile： 能用volatile就不用synchronized。

3、使用线程池： 不要手动创建线程，使用线程池。

4、减少锁竞争： 考虑锁分段、读写分离等优化。

5、避免锁顺序死锁： 固定锁获取顺序。
