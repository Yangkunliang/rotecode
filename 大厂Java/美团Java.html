美团Java社招面试题

1、Java中的HashMap的工作原理是什么？

  • 存储结构：HashMap使用数组+链表/红黑树的结构存储数据。
  • 哈希函数：通过hashCode计算哈希值，再通过取模运算确定数组索引。
  • 冲突处理：使用链地址法处理哈希冲突。
  • 扩容机制：当元素数量超过阈值时，进行扩容操作。
  • 红黑树转换：当链表长度超过8时，转换为红黑树。

2、解释Java中的synchronized关键字

  • 作用：保证多线程环境下的线程安全。
  • 使用方式：可以修饰方法或代码块。
  • 锁类型：分为对象锁和类锁。
  • 锁升级：从偏向锁到轻量级锁再到重量级锁。
  • 性能：JDK 1.6之后进行了优化，性能有所提升。

3、Java中的线程池有哪些类型？

  • FixedThreadPool：固定大小的线程池。
  • CachedThreadPool：可缓存的线程池。
  • ScheduledThreadPool：定时任务线程池。
  • SingleThreadExecutor：单线程线程池。
  • ForkJoinPool：分治任务线程池。

4、解释Java中的内存模型（JMM）

  • 主内存：所有线程共享的内存区域。
  • 工作内存：每个线程私有的内存区域。
  • 可见性：一个线程的修改对其他线程可见。
  • 原子性：操作不可中断。
  • 有序性：指令执行顺序符合程序顺序。

5、Java中的反射机制有什么优缺点？

  • 优点：可以动态加载类、调用方法、访问字段。
  • 优点：提高代码的灵活性和可扩展性。
  • 缺点：性能开销较大。
  • 缺点：破坏封装性，可能访问私有成员。
  • 缺点：代码可读性较差。

6、解释Java中的泛型通配符

  • 无界通配符：使用?表示任意类型。
  • 上界通配符：使用? extends T表示T或其子类。
  • 下界通配符：使用? super T表示T或其父类。
  • 用途：提高方法的灵活性。
  • PECS原则：生产者使用extends，消费者使用super。

7、Java中的Lambda表达式与匿名内部类有什么区别？

  • 语法简洁：Lambda表达式语法更简洁。
  • 类型推断：Lambda表达式支持类型推断。
  • this引用：Lambda表达式中的this指向外部类。
  • 接口限制：Lambda表达式只能用于函数式接口。
  • 性能：Lambda表达式可能更高效。

8、解释Java中的NIO的三大核心组件

  • Channel：双向通道，用于数据传输。
  • Buffer：缓冲区，用于存储数据。
  • Selector：选择器，用于管理多个通道。
  • 非阻塞：支持非阻塞IO操作。
  • 多路复用：单个线程可以处理多个通道。

9、Java中的序列化和反序列化是什么？

  • 序列化：将对象转换为字节流。
  • 反序列化：将字节流转换为对象。
  • 实现方式：实现Serializable接口。
  • 用途：对象持久化、网络传输。
  • 注意事项：serialVersionUID的使用、transient关键字。

10、解释Java中的设计模式

  • 创建型模式：单例、工厂、建造者、原型、抽象工厂。
  • 结构型模式：适配器、桥接、组合、装饰器、外观、享元、代理。
  • 行为型模式：策略、模板方法、观察者、迭代器、责任链等。
  • 原则：开闭原则、依赖倒置原则、单一职责原则等。
  • 应用：在框架和库中广泛使用。

11、Java中的ConcurrentHashMap是如何实现的？

  • 分段锁：将数据分成多个段，每个段独立加锁。
  • CAS操作：使用CAS保证原子性操作。
  • volatile：保证可见性。
  • 扩容机制：支持并发扩容。
  • 性能：高并发场景下性能优异。

12、解释Java中的ThreadLocal

  • 作用：提供线程本地变量。
  • 实现：每个线程有独立的变量副本。
  • 使用场景：数据库连接、Session管理等。
  • 内存泄漏：需要及时清理。
  • InheritableThreadLocal：支持子线程继承。

13、Java中的volatile关键字有什么作用？

  • 可见性：保证变量的修改对所有线程可见。
  • 禁止重排序：确保指令执行顺序。
  • 不保证原子性：复合操作需要额外同步。
  • 使用场景：状态标志、双重检查锁定等。
  • 性能：比普通变量读写慢。

14、解释Java中的类加载机制

  • 加载：读取字节码，创建Class对象。
  • 链接：验证、准备、解析。
  • 初始化：执行静态初始化。
  • 类加载器：Bootstrap、Extension、Application。
  • 双亲委派：委托父类加载器加载。

15、Java中的异常处理机制是怎样的？

  • 异常分类：检查型异常、运行时异常、错误。
  • try-catch-finally：捕获和处理异常。
  • 异常链：包装异常信息。
  • 资源管理：try-with-resources自动关闭资源。
  • 自定义异常：扩展Exception类。

16、解释Java中的多态

  • 定义：同一个行为在不同对象上有不同表现。
  • 编译时多态：方法重载。
  • 运行时多态：方法重写。
  • 向上转型：子类对象赋值给父类引用。
  • 动态绑定：运行时确定调用的方法。

17、Java中的equals和hashCode有什么关系？

  • 约定：相等的对象必须有相等的hashCode。
  • 重写原则：重写equals时必须重写hashCode。
  • 应用：在HashSet和HashMap中使用。
  • 一致性：hashCode在对象不变时应保持一致。
  • 性能：hashCode应均匀分布。

18、解释Java中的AOP

  • 定义：面向切面编程。
  • 核心概念：切面、切点、通知、连接点、织入。
  • 实现方式：动态代理、AspectJ。
  • 应用场景：日志、事务、安全等。
  • 优势：关注点分离、代码复用。

19、Java中的单例模式有哪些实现方式？

  • 饿汉式：类加载时创建实例。
  • 懒汉式：延迟加载。
  • 双重检查锁定：减少同步开销。
  • 静态内部类：利用类加载机制。
  • 枚举：最安全的方式。

20、解释Java中的垃圾回收机制

  • 标记-清除：标记并清除无用对象。
  • 复制算法：复制存活对象到新空间。
  • 标记-整理：标记后整理内存。
  • 分代收集：根据对象存活时间处理。
  • 垃圾收集器：Serial、Parallel、CMS、G1等。
