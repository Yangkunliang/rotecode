小米Java社招面试题

1、Java中的HashMap和HashTable有什么区别？

  • 线程安全：HashMap非线程安全，HashTable线程安全。
  • null值：HashMap允许null键值，HashTable不允许。
  • 性能：HashMap性能更好。
  • 继承：HashMap继承AbstractMap，HashTable继承Dictionary。
  • 迭代器：HashMap的迭代器是fail-fast的。

2、解释Java中的线程池

  • 组成：核心线程、任务队列、最大线程、拒绝策略。
  • 工作流程：提交任务→核心线程处理→队列等待→扩容→拒绝。
  • 类型：FixedThreadPool、CachedThreadPool等。
  • 优势：线程复用、控制并发、管理任务。
  • 监控：ThreadPoolExecutor的监控方法。

3、Java中的synchronized和ReentrantLock有什么区别？

  • 实现：synchronized是关键字，ReentrantLock是类。
  • 锁释放：synchronized自动释放，ReentrantLock需手动释放。
  • 功能：ReentrantLock支持公平锁、可中断等。
  • 性能：JDK 1.6后两者性能相近。
  • 使用场景：复杂场景用ReentrantLock。

4、解释Java内存模型

  • 主内存：共享变量存储区域。
  • 工作内存：线程私有缓存。
  • 可见性：volatile保证可见性。
  • 原子性：synchronized/CAS保证。
  • 有序性：happens-before规则。

5、Java中的泛型是如何工作的？

  • 类型擦除：编译时移除泛型信息。
  • 编译检查：编译时进行类型检查。
  • 桥接方法：保持多态性。
  • 通配符：?、extends、super。
  • 优点：类型安全、代码复用。

6、解释Java中的反射机制

  • 定义：运行时访问类的信息。
  • 用途：动态加载类、调用方法、访问字段。
  • API：Class、Method、Field、Constructor。
  • 性能：比直接调用慢。
  • 安全性：可能破坏封装。

7、Java中的Lambda表达式是什么？

  • 定义：简洁的匿名函数。
  • 语法：(参数) -> 表达式。
  • 函数式接口：单一抽象方法。
  • 方法引用：简化Lambda。
  • 优点：代码简洁、支持并行。

8、解释Java中的NIO

  • 非阻塞：线程不阻塞等待IO。
  • Channel：双向数据通道。
  • Buffer：数据缓冲区。
  • Selector：多路复用器。
  • 应用：高并发网络编程。

9、Java中的序列化机制

  • 定义：对象转字节流。
  • 实现：Serializable接口。
  • serialVersionUID：版本控制。
  • transient：跳过序列化。
  • 用途：持久化、网络传输。

10、解释Java中的设计模式

  • 创建型：单例、工厂、建造者、原型。
  • 结构型：适配器、装饰器、代理。
  • 行为型：策略、观察者、迭代器。
  • 原则：开闭、单一职责、依赖倒置。
  • 应用：提高代码质量。

11、Java中的ThreadLocal是什么？

  • 作用：提供线程本地变量。
  • 实现：ThreadLocalMap存储。
  • 使用场景：数据库连接、Session。
  • 内存泄漏：弱引用+及时清理。
  • 继承：InheritableThreadLocal。

12、解释Java中的volatile关键字

  • 可见性：保证变量修改对所有线程可见。
  • 禁止重排序：内存屏障。
  • 不保证原子性：复合操作需同步。
  • 使用场景：状态标志、双重检查。
  • 性能：比普通变量慢。

13、Java中的类加载机制

  • 加载：读取字节码创建Class对象。
  • 链接：验证、准备、解析。
  • 初始化：执行静态代码块。
  • 类加载器：Bootstrap、Extension、Application。
  • 双亲委派：委托父类加载器。

14、解释Java中的异常处理

  • 分类：检查型、运行时、错误。
  • try-catch-finally：捕获处理。
  • 异常链：包装异常信息。
  • 资源管理：try-with-resources。
  • 自定义异常：扩展Exception。

15、Java中的多态是如何实现的？

  • 编译时：方法重载。
  • 运行时：方法重写。
  • 向上转型：子类赋值给父类。
  • 动态绑定：运行时确定方法。
  • 优点：提高代码灵活性。

16、解释Java中的equals和hashCode

  • equals：比较对象内容。
  • hashCode：返回哈希码。
  • 约定：相等对象hashCode必须相等。
  • 重写：两者需同时重写。
  • 应用：HashSet、HashMap。

17、Java中的AOP是如何实现的？

  • 动态代理：JDK动态代理或CGLIB。
  • AspectJ：编译时或运行时织入。
  • 核心概念：切面、切点、通知。
  • 应用场景：日志、事务、安全。
  • 优点：关注点分离。

18、解释Java中的单例模式

  • 定义：保证类只有一个实例。
  • 实现：饿汉式、懒汉式、双重检查。
  • 线程安全：需考虑多线程。
  • 序列化：需重写readResolve。
  • 枚举：最安全的实现。

19、Java中的垃圾回收机制

  • 算法：标记-清除、复制、标记-整理。
  • 分代：年轻代、老年代、元空间。
  • 收集器：Serial、Parallel、CMS、G1。
  • 调优：堆大小、收集器选择。
  • 监控：jstat、jmap。

20、解释Java中的wait和notify机制

  • wait：线程等待并释放锁。
  • notify：唤醒单个线程。
  • notifyAll：唤醒所有线程。
  • 使用：必须在同步块中。
  • 场景：生产者-消费者模式。
