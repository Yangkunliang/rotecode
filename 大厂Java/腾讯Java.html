腾讯Java社招面试题

1、Java中的HashMap是如何实现的？

  • 存储结构：数组+链表/红黑树。
  • 哈希函数：hashCode()和扰动函数。
  • 冲突处理：链地址法。
  • 扩容机制：2倍扩容，重新哈希。
  • 红黑树：链表长度>8时转换。

2、解释Java中的线程池

  • 核心参数：核心线程数、最大线程数、队列、拒绝策略。
  • 工作流程：提交任务→核心线程执行→队列等待→扩容线程→拒绝。
  • 拒绝策略：Abort、CallerRuns、Discard、DiscardOldest。
  • 类型：Fixed、Cached、Scheduled、SingleThread。
  • 优势：线程复用、控制并发数、管理任务。

3、Java中的synchronized是如何实现的？

  • 对象头：Mark Word存储锁信息。
  • 锁升级：偏向锁→轻量级锁→重量级锁。
  • monitor：操作系统互斥锁。
  • wait/notify：通过monitor实现。
  • 性能优化：锁消除、锁粗化。

4、解释Java内存模型（JMM）

  • 主内存：共享变量存储区域。
  • 工作内存：线程私有缓存。
  • 可见性：volatile保证可见性。
  • 原子性：synchronized/CAS保证。
  • 有序性：happens-before规则。

5、Java中的泛型是什么？

  • 定义：参数化类型，提高代码复用性。
  • 类型擦除：编译时移除泛型信息。
  • 通配符：?、extends、super。
  • PECS原则：生产者extends，消费者super。
  • 优点：类型安全、避免强制转换。

6、解释Java中的反射机制

  • 定义：运行时获取类信息和调用方法。
  • 用途：框架开发、动态代理、测试。
  • API：Class、Method、Field、Constructor。
  • 性能：比直接调用慢。
  • 安全性：需权限检查。

7、Java中的Lambda表达式

  • 定义：简洁的函数式接口实现。
  • 语法：(参数) -> {体}。
  • 函数式接口：单一抽象方法。
  • 方法引用：简化Lambda。
  • 优点：代码简洁、支持并行。

8、解释Java中的NIO

  • 非阻塞：线程不阻塞等待IO。
  • Channel：双向数据通道。
  • Buffer：数据容器。
  • Selector：多路复用器。
  • 应用：高并发网络编程。

9、Java中的序列化机制

  • 定义：对象转字节流。
  • 实现：Serializable接口。
  • serialVersionUID：版本控制。
  • transient：跳过序列化。
  • Externalizable：自定义序列化。

10、解释Java中的设计模式

  • 创建型：单例、工厂、建造者、原型。
  • 结构型：适配器、装饰器、代理、组合。
  • 行为型：策略、观察者、迭代器、责任链。
  • 原则：开闭、单一职责、依赖倒置。
  • 应用：提高代码质量和可维护性。

11、Java中的ThreadLocal

  • 作用：线程本地变量。
  • 实现：ThreadLocalMap存储。
  • 使用场景：数据库连接、Session。
  • 内存泄漏：弱引用+及时清理。
  • 继承：InheritableThreadLocal。

12、解释Java中的volatile

  • 可见性：保证变量修改对所有线程可见。
  • 禁止重排序：内存屏障。
  • 不保证原子性：复合操作需同步。
  • 使用场景：状态标志、双重检查。
  • 性能：比普通变量慢。

13、Java中的类加载机制

  • 加载：读取字节码创建Class对象。
  • 链接：验证、准备、解析。
  • 初始化：执行静态代码块。
  • 类加载器：Bootstrap、Extension、Application。
  • 双亲委派：委托父类加载器。

14、解释Java中的异常处理

  • 分类：检查型、运行时、错误。
  • try-catch-finally：捕获处理。
  • 异常链：包装异常信息。
  • 资源管理：try-with-resources。
  • 自定义异常：扩展Exception。

15、Java中的多态

  • 编译时：方法重载。
  • 运行时：方法重写。
  • 向上转型：子类赋值给父类。
  • 动态绑定：运行时确定方法。
  • 优点：提高代码灵活性。

16、解释Java中的equals和hashCode

  • equals：比较对象内容。
  • hashCode：返回哈希码。
  • 约定：相等对象hashCode必须相等。
  • 重写：两者需同时重写。
  • 应用：HashSet、HashMap。

17、Java中的AOP

  • 定义：面向切面编程。
  • 核心概念：切面、切点、通知。
  • 实现：动态代理、AspectJ。
  • 应用：日志、事务、安全。
  • 优点：关注点分离。

18、解释Java中的单例模式

  • 定义：保证类只有一个实例。
  • 实现：饿汉式、懒汉式、双重检查。
  • 线程安全：需考虑多线程。
  • 序列化：需重写readResolve。
  • 枚举：最安全的实现。

19、Java中的垃圾回收

  • 算法：标记-清除、复制、标记-整理。
  • 分代：年轻代、老年代、永久代/元空间。
  • 收集器：Serial、Parallel、CMS、G1。
  • 调优：堆大小、收集器选择。
  • 监控：jstat、jmap、jvisualvm。

20、解释Java中的wait和notify

  • wait：线程等待并释放锁。
  • notify：唤醒单个线程。
  • notifyAll：唤醒所有线程。
  • 使用：必须在同步块中。
  • 场景：生产者-消费者模式。
