MySQL 面试题及答案整理

1. InnoDB 与 MyISAM 存储引擎的区别

特性InnoDBMyISAM
事务支持✅ 支持❌ 不支持
锁粒度行级锁表级锁
外键约束✅ 支持❌ 不支持
全文索引❌ 不支持（5.6前）✅ 支持
崩溃恢复✅ 自动恢复❌ 较脆弱
存储限制受文件系统限制支持较大文件
其他区别：
  • InnoDB 支持 MVCC，适用于高并发场景
  • MyISAM 适用于只读或少量写操作的场景

3. MySQL 高效分页查询

分页优化方法

  1. 避免大 OFFSET
    • LIMIT 100000, 10 性能差
    • 改用 WHERE id > last_id LIMIT 10
  2. 使用覆盖索引 - 查询只访问索引，不回表
  3. 利用查询缓存 - 适用于数据变化不频繁的场景
  4. 记录上次位置 - 记住上次查询的最后位置，而不是使用 OFFSET

4. MySQL 锁机制

锁的分类

  • 表级锁：锁定整个表，开销小，并发低
  • 行级锁：锁定单行数据，开销大，并发高
  • 页级锁：介于表级和行级之间

InnoDB 锁类型

  • 共享锁 (S Lock)：允许读，阻止写
  • 排他锁 (X Lock)：允许写，阻止其他锁
  • 意向锁：表级锁，表示事务准备在某行上加锁

5. 事务隔离级别

隔离级别脏读不可重复读幻读
READ UNCOMMITTED❌ 可能❌ 可能❌ 可能
READ COMMITTED✅ 防止❌ 可能❌ 可能
REPEATABLE READ✅ 防止✅ 防止❌ 可能
SERIALIZABLE✅ 防止✅ 防止✅ 防止

MySQL 默认隔离级别为 REPEATABLE READ。

6. MySQL 死锁处理

死锁产生条件

  • 互斥条件：资源不可共享
  • 请求与保持条件：持有资源并请求新资源
  • 不剥夺条件：资源不可强制剥夺
  • 循环等待条件：形成等待环路

死锁解决方法

  • 按顺序访问资源
  • 设置锁超时时间
  • 使用死锁检测与回滚
  • 减少事务持有锁的时间

7. ACID 特性

  • 原子性 (Atomicity)：事务是原子的，要么全部执行，要么全部不执行
  • 一致性 (Consistency)：事务执行前后数据库状态一致
  • 隔离性 (Isolation)：事务之间相互隔离
  • 持久性 (Durability)：事务提交后数据持久保存

8. 索引覆盖

覆盖索引：查询所需的所有列都在索引中，无需回表查询。

优势：减少磁盘 I/O，提高查询性能

适用场景：频繁执行的查询，列较少的查询。

9. GROUP BY 与 ORDER BY

GROUP BY 优化

  • 使用索引避免排序
  • 尽量在索引列上进行 GROUP BY
  • 减少 GROUP BY 的列数

ORDER BY 优化

  • 使用索引避免排序
  • 避免在 ORDER BY 中使用函数
  • 减少排序的数据量

10. 慢查询日志

慢查询日志配置

  • slow_query_log = ON - 开启慢查询日志
  • long_query_time = 2 - 设置慢查询阈值（秒）
  • log_queries_not_using_indexes = ON - 记录未使用索引的查询

慢查询分析工具

  • mysqldumpslow：MySQL 自带的慢查询日志分析工具
  • pt-query-digest：Percona Toolkit 中的工具

11. JOIN 类型

JOIN 类型说明
INNER JOIN返回两表匹配的行
LEFT JOIN返回左表所有行，右表匹配的行
RIGHT JOIN返回右表所有行，左表匹配的行
FULL JOIN返回两表所有行
CROSS JOIN笛卡尔积

12. 二级索引

二级索引：除主键索引外的其他索引。

  • InnoDB 的二级索引叶子节点存储主键值
  • 通过二级索引查询需要回表（除非覆盖索引）
  • 二级索引可以加速查询，但会降低写性能

13. MVCC 机制

MVCC (Multi-Version Concurrency Control)：多版本并发控制。

工作原理

  • 每行记录有两个隐藏列：trx_id（事务ID）和 roll_pointer（回滚指针）
  • 事务读取时根据可见性规则判断数据版本
  • Undo 日志存储旧版本数据

14. MySQL 高可用架构

  • 主从复制：一主多从，读负载均衡
  • MGR (MySQL Group Replication)：多主复制，自动故障转移
  • 半同步复制：至少一个从库确认后才提交
  • ProxySQL：数据库中间件，读写分离

15. 二阶段提交 (2PC)

两阶段

  1. 准备阶段：协调者询问所有参与者是否可以提交
  2. 提交阶段：协调者根据反馈决定提交或回滚
问题：协调者故障会导致数据不一致

16. 视图

视图：基于 SQL 查询的虚拟表。

视图的作用

  • 简化复杂查询
  • 数据安全（限制访问）
  • 逻辑独立性
注意：视图不存储数据，每次访问都会执行查询

17. 大表优化

  • 分表：水平分表、垂直分表
  • 分区：按范围、列表、哈希分区
  • 索引优化：合理创建索引
  • 读写分离：主库写，从库读
  • 缓存：使用 Redis 缓存热点数据

18. 存储过程与函数

存储过程

  • 预编译的 SQL 语句集合
  • 支持流程控制（IF、WHILE 等）
  • 可返回多个结果集

函数

  • 必须返回单个值
  • 可在 SQL 语句中调用

19. 查询优化方法

  • 使用 EXPLAIN 分析执行计划
  • 避免 SELECT *
  • 合理使用索引
  • 避免在 WHERE 子句中使用函数
  • 使用连接代替子查询
  • 限制返回行数
