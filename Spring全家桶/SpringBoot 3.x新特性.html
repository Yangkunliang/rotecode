    SpringBoot 3.x新特性

    Spring Boot 3.0有哪些重大变化？

    1. 最低JDK版本提升：Spring Boot 3.0要求最低JDK 17，不再支持JDK 8和11。
    2. Jakarta EE迁移：从Java EE迁移到Jakarta EE API，javax.*包名全部改为jakarta.*。
    3. GraalVM原生镜像支持：更好的支持GraalVM，可以将应用编译成原生镜像，大幅提升启动速度并降低内存占用。
    4. Spring 6框架升级：基于Spring Framework 6.0，引入了更多的响应式编程特性和改进。
    5. Micrometer 2.0：升级到Micrometer 2.0，提供更强大的指标监控能力。

    Spring Boot 3.x对Jakarta EE的迁移具体包含哪些内容？

    1. 包名变更：所有javax.*命名空间的类迁移到jakarta.*命名空间。
    2. 影响范围：包括Servlet、JDBC、JPA、Bean Validation等API的迁移。
    3. 第三方库兼容：需要确保项目使用的第三方库也支持Jakarta EE。
    4. 代码修改：现有的代码和XML配置中的javax.包引用需要更新为jakarta.。

    Spring Boot 3.x中GraalVM原生镜像的支持带来了哪些优势？

    1. 极速启动：原生镜像的启动时间可以从几秒降低到毫秒级别。
    2. 内存占用低：相比JVM，内存占用大幅减少，适合Serverless和容器化部署。
    3. 即时编译：无需预热，编译好的原生代码可以立即以最高性能运行。
    4. AOT编译：通过Spring Boot的AOT（Ahead-of-Time）处理，优化应用的启动和运行时性能。

    Spring Boot 3.x中的WebFlux有什么新特性？

    1. 响应式Web框架：继续强化WebFlux响应式编程模型。
    2. 更好的性能：在Spring 6的基础上，WebFlux的性能和稳定性都有提升。
    3. 简化配置：自动配置更加智能，减少了响应式应用的配置复杂度。

    Spring Boot 3.x的AOT编译是如何工作的？

    1. AOT处理：在构建阶段进行的编译优化，可以提前生成优化的字节码和原生代码。
    2. GraalVM集成：通过GraalVM将Spring Boot应用编译成原生镜像。
    3. 反射信息处理：AOT阶段会处理应用的反射使用情况，生成必要的配置文件。
    4. 启动优化：原生镜像消除了JVM启动时的类加载和即时编译开销。

    Spring Boot 3.x如何提升了对容器化部署的支持？

    1. 分层镜像：支持更高效的分层Docker镜像构建。
    2. 健康检查优化：提供更精确的健康检查端点。
    3. 环境变量绑定：增强了环境变量到配置属性的绑定。
    4. 原生镜像友好：配合GraalVM，容器内运行更加轻量。

    Spring Boot 3.x的安全特性有哪些改进？

    1. Spring Security 6.0：配合Spring Security 6.0，提供更细粒度的安全控制。
    2. 简化配置：安全配置的自动配置更加智能。
    3. Jakarta EE支持：安全相关的API也迁移到jakarta.*命名空间。

    Spring Boot 3.x中的自动配置有哪些改进？

    1. 条件注解优化：自动配置使用的条件注解更加精确。
    2. 性能提升：自动配置的处理过程更加高效。
    3. 可观测性：自动配置过程提供更好的可观测性，便于调试。

    Spring Boot 3.x如何处理观测性（Observability）方面的增强？

    1. Micrometer 2.0：升级到Micrometer 2.0，提供更丰富的指标收集能力。
    2. 统一结构化日志：支持统一的日志结构，便于分析和追踪。
    3. 健康检查端点：增强的健康检查和数据展示。
    4. 分布式追踪：更好地与Zipkin、Jaeger等追踪系统集成。

    从Spring Boot 2.x升级到3.x需要注意哪些事项？

    1. JDK版本：必须升级到JDK 17或更高版本。
    2. Jakarta EE迁移：所有javax.*需要改为jakarta.*。
    3. 依赖检查：检查所有第三方依赖是否支持Spring Boot 3.x和Jakarta EE。
    4. 配置迁移：检查并更新任何特定的Spring Boot 2.x配置。
    5. 测试验证：充分测试应用的各个方面，确保兼容性。

    Spring Boot 3.x中DevTools有哪些改进？

    1. 更快的重启：通过更智能的类加载策略，实现更快速的应用重启。
    2. .LiveReload增强：页面热重载更加稳定和快速。
    3. 调试支持：改进了远程调试的体验。

    Spring Boot 3.x对数据访问层有哪些增强？

    1. JDBC增强：JDBC操作支持更多的响应式特性。
    2. R2DBC支持：更好地支持R2DBC进行响应式数据库访问。
    3. Jakarta EE迁移：JPA和Hibernate等已迁移到Jakarta EE。
