    SpringData JPA

    Spring Data JPA的工作原理是什么？

    1. Repository接口：开发者只需定义继承JpaRepository的接口，Spring Data JPA会根据接口自动生成实现类。
    2. 方法名解析：Spring Data JPA会根据方法名自动解析查询逻辑，如findByUsername会生成根据username查询的JPQL。
    3. JPA Provider：底层使用Hibernate或其他JPA Provider来实现ORM映射。
    4. 实体管理：Spring Data JPA通过EntityManager管理实体对象，实现持久化操作。
    通过这些机制，Spring Data JPA大幅简化了数据访问层的代码编写。

    Spring Data JPA中Repository接口的层次结构是怎样的？

    1. Repository：最基础的标记接口，没有任何方法。
    2. CrudRepository：提供了基本的CRUD操作方法。
    3. PagingAndSortingRepository：提供了分页和排序的方法。
    4. JpaRepository：在PagingAndSortingRepository基础上添加了JPA特有的方法，如flush()和batch()操作。
    5. 自定义Repository：开发者可以定义自己的接口和方法，Spring Data JPA会自动实现。

    Spring Data JPA如何实现自定义查询？

    1. 方法名查询：通过规范的方法名自动生成查询，如findByUsername。
    2. @Query注解：使用JPQL或原生SQL进行自定义查询。
    3. @Query with LIKE：在@Query中使用LIKE进行模糊查询。
    4. 使用Specification：Spring Data JPA提供了JpaSpecificationExecutor，支持动态查询。
    5. 自定义实现：定义一个接口和对应的实现类，在实现类中编写复杂的查询逻辑。

    Spring Data JPA中实体之间的关系如何映射？

    1. @OneToOne：一对一关系映射。
    2. @OneToMany和@ManyToOne：一对多和多对一关系。
    3. @ManyToMany：多对多关系映射。
    4. 级联操作：通过cascade属性配置级联保存、删除等行为。
    5. 懒加载：通过fetch属性配置加载策略，如FetchType.LAZY。

    Spring Data JPA中如何实现分页和排序？

    1. PagingAndSortingRepository：继承这个接口即可获得分页和排序支持。
    2. PageRequest：使用PageRequest.of(page, size)创建分页请求。
    3. Sort：使用Sort.by()创建排序规则。
    4. Page对象：返回的Page对象包含了分页信息和查询结果。

    Spring Data JPA中@Entity注解的作用是什么？

    1. 标记为实体：@Entity注解标记一个类为JPA实体，表示该类对应数据库中的一张表。
    2. 表名映射：默认情况下，实体名对应表名，也可以通过@Table注解指定具体表名。
    3. 生命周期管理：被@Entity注解的类会被EntityManager管理，支持CRUD操作。

    Spring Data JPA中如何处理实体主键的生成策略？

    1. @GeneratedValue：配合@Id注解使用，配置主键的生成策略。
    2. AUTO：由JPA自动选择合适的策略。
    3. IDENTITY：由数据库自增生成，如MySQL的自增主键。
    4. SEQUENCE：使用数据库序列生成，如Oracle。
    5. TABLE：使用一张表来生成主键值。

    Spring Data JPA中的事务管理是如何工作的？

    1. @Transactional注解：在Repository或Service层方法上使用此注解来声明事务。
    2. 默认传播行为：Spring Data JPA的事务默认传播行为是REQUIRED。
    3. 只读事务：可以通过readOnly=true优化只读操作。
    4. 回滚规则：默认情况下，运行时异常会自动回滚。

    Spring Data JPA中如何实现懒加载？有什么需要注意的？

    1. fetch = FetchType.LAZY：在关联映射上使用此配置实现懒加载。
    2. Open Session in View：需要配置Open Session in View模式以避免懒加载异常。
    3. 懒加载异常：如果在session关闭后访问懒加载属性，会抛出LazyInitializationException。
    4. 解决方案：可以使用@EntityGraph或JOIN FETCH来明确加载关联数据。

    Spring Data JPA与MyBatis的区别是什么？

    1. ORM vs SQL映射：JPA是ORM框架，MyBatis是SQL映射框架。
    2. 开发方式：JPA自动生成SQL，MyBatis需要手写SQL。
    3. 灵活性：MyBatis对复杂查询和SQL优化更灵活，JPA开发更快捷。
    4. 数据库无关：JPA提供更好的数据库无关性。

    Spring Data JPA中如何实现条件查询？

    1. 使用方法名查询：如findByUsernameAndEmail()。
    2. 使用@Query：自定义JPQL或原生SQL查询。
    3. 使用Specification：JpaSpecificationExecutor提供了强大的动态查询能力。
    4. Example查询：使用Example对象进行动态查询。

    Spring Data JPA中的@Modifying注解有什么作用？

    1. 标记修改操作：@Modifying注解用于标记修改、删除等更新操作。
    2. 配合@Query使用：通常与@Query一起使用来执行更新或删除操作。
    3. 事务要求：使用@Modifying的方法必须开启事务。
    4. 清除缓存：执行修改操作后，Spring Data JPA会自动清除相关的持久化缓存。

    Spring Data JPA中如何处理实体类的乐观锁？

    1. @Version注解：在实体类的版本字段上使用@Version注解。
    2. 自动版本控制：每次更新时，版本号自动递增。
    3. 乐观锁异常：如果并发更新，Spring Data JPA会抛出OptimisticLockingFailureException。
    4. 适用场景：适用于读多写少的场景，避免数据库级别的锁。
