Hive 面试题及答案整理

1. Hive的基本架构是怎样的？

Hive是一个数据仓库基础设施，其基本架构包括：

1、用户接口： CLI、WebUI、JDBC/ODBC。

2、Thrift服务： 提供跨语言支持。

3、Compiler： 解析SQL，生成执行计划。

4、Optimizer： 优化执行计划。

5、Execution Engine： 执行MapReduce/Tez/Spark任务。

2. Hive的数据存储模型是怎样的？

Hive的数据存储结构：

1、Database： 命名空间，类似关系型数据库。

2、Table： 表，分为内部表和外部表。

3、Partition： 分区，提高查询效率。

4、Bucket： 分桶，进一步细化数据。

3. Hive的内部表和外部表的区别？

内部表和外部表的区别：

1、存储位置： 内部表数据存储在Hive仓库，外部表指定位置。

2、删除行为： 删除内部表时数据一并删除，外部表只删除元数据。

3、使用场景： 内部表用于临时数据，外部表用于共享数据。

4、创建语法： 外部表使用EXTERNAL关键字。

4. Hive的分区和分桶有什么区别？

分区和分桶的区别：

1、分区： 按列值划分目录，提高查询效率。

2、分桶： 按hash值划分文件，支持采样和JOIN优化。

3、粒度： 分区粒度较粗，分桶粒度更细。

4、创建： 分区在INSERT时自动创建，分桶需要先创建表结构。

5. Hive的SQL执行流程是怎样的？

Hive的SQL执行流程：

1、解析： ANTLR解析SQL生成AST。

2、语义分析： 验证表、列是否存在。

3、生成执行计划： 生成逻辑执行计划和物理执行计划。

4、优化： Predicate pushdown、Partition pruning等优化。

5、执行： 提交到MapReduce/Tez/Spark执行。

6. Hive的UDF有哪几种类型？

Hive的UDF类型：

1、UDF： 一对一输入输出，如UPPER、LOWER。

2、UDAF： 多对一输入输出，如SUM、COUNT。

3、UDTF： 一对多输入输出，如EXPLODE。

4 GenericUDF： 泛型UDF，支持复杂类型。

7. Hive如何进行优化？

Hive优化的主要方向：

1、SQL优化： 避免SELECT *、合理使用WHERE。

2、分区裁剪： 开启分区裁剪减少数据量。

3、JOIN优化： 大表放右边、使用MAP JOIN。

4、数据倾斜： 处理数据倾斜问题。

5、并行执行： 开启并行执行。

8. Hive的数据倾斜如何解决？

Hive数据倾斜的解决方案：

1、JOIN倾斜： 大表JOIN小表，使用MAP JOIN。

2、GROUP BY倾斜： 开启倾斜处理参数。

3、加盐： 给热点key加随机前缀。

4、单独处理： 将倾斜的key单独处理。

9. Hive的MapJoin原理是什么？

MapJoin的原理：

1、原理： 将小表加载到内存，在Map端完成JOIN。

2、优势： 避免Shuffle和Reduce阶段。

3、触发条件： 小表大小不超过阈值。

4、配置： 设置hive.auto.convert.join为true。

10. Hive的窗口函数有哪些？

Hive支持的窗口函数：

1、聚合窗口函数： SUM、AVG、COUNT、MAX、MIN。

2、排名函数： ROW_NUMBER、RANK、DENSE_RANK、CUME_DIST。

3、分析函数： LEAD、LAG、FIRST_VALUE、LAST_VALUE。

4、窗口子句： OVER()、PARTITION BY、ORDER BY。

11. Hive的SerDe是什么？

Hive的SerDe机制：

1、概念： Serializer/Deserializer，序列化/反序列化。

2、作用： 读取数据时反序列化，写数据时序列化。

3、内置： LazySimpleSerDe、ORCSerDe、ParquetSerDe。

4、自定义： 可以实现Custom SerDe。

12. Hive支持哪些文件格式？

Hive支持的文件格式：

1、TextFile： 文本格式，默认格式。

2、SequenceFile： 二进制序列文件。

3、RCFile： 列式存储文件。

4、ORC： 优化的列式存储，性能更好。

5、Parquet： 列式存储，跨语言支持好。

13. Hive的元数据存储在哪里？

Hive元数据存储：

1、默认： 内置Derby数据库，单用户。

2、生产： 使用MySQL、PostgreSQL存储元数据。

3、远程模式： Metastore服务独立部署。

4、配置： 通过hive.metastore.uris配置。

14. Hive的动态分区和静态分区有什么区别？

动态分区和静态分区的区别：

1、静态分区： 分区值在SQL中指定。

2、动态分区： 分区值从数据中获取。

3、配置： 需要开启hive.exec.dynamic.partition。

4、效率： 动态分区更灵活，静态分区更高效。

15. Hive的COUNT(DISTINCT)有什么问题？

COUNT(DISTINCT)的问题：

1、问题： 所有数据会经过一个Reduce处理。

2、优化： 先用GROUP BY去重，再COUNT。

3、示例： SELECT COUNT(DISTINCT key) FROM t → SELECT COUNT(*) FROM (SELECT key FROM t GROUP BY key) t。

4、原因： DISTINCT需要全局排序。

16. Hive如何处理NULL值？

Hive处理NULL值的方式：

1、存储： NULL存储为空字符串或特定值。

2、查询： 使用IS NULL和IS NOT NULL判断。

3、函数： 使用NVL、COALESCE处理NULL。

4、空字符串： 使用''表示空字符串。

17. Hive的Lateral View是什么？

Lateral View用于和UDTF配合：

1、作用： 将UDTF的结果展开为多行。

2、示例： 与EXPLODE配合拆分数组。

3、语法： LATERAL VIEW udtf(expr) tableAlias AS colAlias。

4、多视图： 可以有多个Lateral View。

18. Hive的SORT BY、ORDER BY、DISTRIBUTE BY有什么区别？

这三种排序的区别：

1、ORDER BY： 全局排序，只有一个Reducer。

2、SORT BY： 每个Reducer内排序，不保证全局顺序。

3、DISTRIBUTE BY： 控制数据分发到哪个Reducer。

4、CLUSTER BY： DISTRIBUTE BY + SORT BY的简写。

19. Hive支持事务吗？

Hive事务支持情况：

1、支持情况： Hive从0.14开始支持ACID事务。

2、限制： 需要使用ORC格式，支持UPDATE和DELETE。

3、配置： 开启hive.txn.manager。

4、场景： 适用于小批量数据更新。

20. Hive如何进行权限管理？

Hive的权限管理：

1、存储授权： 基于HDFS的存储权限。

2、SQL标准授权： GRANT、REVOKE、CREATE ROLE。

3、配置： 开启hive.security.authorization.enabled。

4、粒度： 支持库、表、列级别的权限控制。

21. Hive的JOIN优化有哪些？

Hive JOIN优化的方法：

1、MAP JOIN： 小表放内存，Map端完成JOIN。

2、BUCKET MAP JOIN： 分桶表的MAP JOIN优化。

3、SMB JOIN： 分桶大表间的Sort Merge Bucket JOIN。

4、JOIN顺序： 大表放最后，右边放大表。

22. Hive的数据压缩格式有哪些？

Hive支持的数据压缩格式：

1、Gzip： 压缩比高，不支持切片。

2、Bzip2： 压缩比最高，支持切片。

3、Snappy： 压缩速度快，不支持切片。

4、Lzo： 压缩速度块，支持切片。

23. Hive的执行引擎有哪些？

Hive的执行引擎：

1、MapReduce： 传统引擎，较慢。

2、Tez： 基于DAG，消除了中间结果写磁盘。

3、Spark： 基于内存计算，性能最好。

4、配置： 通过hive.execution.engine设置。

24. Hive的倾斜join优化参数有哪些？

Hive倾斜JOIN优化参数：

1、hive.optimize.skewjoin： 开启倾斜JOIN优化。

2、hive.skewjoin.key： 设定倾斜key的阈值。

3、hive.skewjoin.mapjoin.map.tasks： 倾斜key的Map Join任务数。

4、hive.skewjoin.mapjoin.min.split： 最小Split大小。

25. Hive如何处理小文件问题？

Hive小文件问题处理方法：

1、合并输入： 设置hive.merge.mapfiles和hive.merge.mapredfiles。

2、合并输出： 设置hive.merge.size.per.task。

3、Bucket： 使用分桶减少小文件。

4、SequenceFile： 使用SequenceFile格式。

26. Hive的虚拟列是什么？

Hive虚拟列：

1、INPUT__FILE__NAME： 输入文件名。

2、BLOCK__FILE__OFFSET： 行在文件中的偏移量。

3、ROW__OFFSET： 行在块中的偏移量。

4、使用： 用于数据质量检查和调试。

27. Hive的Fetch Task是什么？

Fetch Task机制：

1、概念： 简单的查询直接读取数据，不启动MR。

2、配置： hive.fetch.task.conversion参数控制。

3、阈值： 可以设置阈值决定是否启用。

4、场景： SELECT、WHERE、LIMIT等简单查询。

28. Hive的列式存储有哪些优势？

列式存储的优势：

1、查询效率： 只读取需要的列，减少IO。

2、压缩效率： 同列数据类型一致，压缩率高。

3、代表格式： ORC、Parquet是常用列式格式。

4、适用场景： OLAP场景，复杂分析查询。

29. Hive的严格模式是什么？

Hive严格模式：

1、配置： hive.mapred.mode=strict。

2、限制： 禁止全表扫描、ORDER BY需要LIMIT。

3、分区表： 禁止对分区分区表执行静态分区。

4、目的： 防止消耗过多资源的查询。

30. Hive如何进行性能监控？

Hive性能监控方式：

1、Web UI： 查看Job执行情况。

2、EXPLAIN： 查看SQL执行计划。

3、History Server： 查看历史Job。

4、Metrics： Hadoop Metrics收集性能指标。

31. Hive的CTE（公共表表达式）如何使用？

CTE（Common Table Expression）：

1、语法： WITH cte AS (SELECT ...) SELECT ... FROM cte。

2、递归： 支持递归CTE用于树形结构。

3、多次引用： CTE结果可在主查询中多次引用。

4、替代： 可以替代子查询，使SQL更易读。

32. Hive的索引有哪些类型？

Hive索引类型：

1、compact索引： 存储相同值的row IDs。

2、bitmap索引： 使用bitmap存储，适用于低基数字段。

3、创建索引： CREATE INDEX idx ON TABLE col。

4、自动使用： 索引需要手动重建和启用。

33. Hive的空值处理函数有哪些？

Hive的空值处理函数：

1、NVL： NVL(expr1, expr2)，expr1为NULL时返回expr2。

2、COALESCE： 返回第一个非NULL值。

3、IFNULL： MySQL兼容的NULL处理。

4、CASE WHEN： 使用条件表达式处理NULL。

34. Hive的桶式采样和块采样有什么区别？

两种采样方式的区别：

1、Bucket Sample： 按桶采样，基于hash。

2、Block Sample： 按数据块采样，按行数百分比。

3、语法： TABLESAMPLE(BUCKET x OUT OF y)。

4、用途： 用于数据探索和测试。

35. Hive 3.0有哪些新特性？

Hive 3.0的新特性：

1、LLAP： Long Live Process and Cache，提升查询性能。

2、ACID 2.0： 改进的事务支持。

3、视图物化： 支持物化视图。

4、优化器改进： Cost-based Optimizer改进。
