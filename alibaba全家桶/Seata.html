    Seata

    Seata是什么？它主要用于解决什么问题？

    1. 分布式事务解决方案：Seata是阿里巴巴开源的分布式事务解决方案，致力于在微服务架构下提供高性能和简单易用的分布式事务服务。
    2. AT模式：提供AT模式，对业务代码无侵入，自动化处理分布式事务。
    3. 多种事务模式：支持AT、TCC、SAGA和XA四种事务模式。
    4. 高可用：支持高可用集群部署，保证事务服务的稳定性。
    Seata通过提供一站式的分布式事务解决方案，大大简化了微服务架构下的事务管理。

    Seata的核心组件有哪些？

    1. Transaction Coordinator (TC)：事务协调器，维护全局事务的运行状态，负责协调并驱动全局事务的提交或回滚。
    2. Transaction Manager (TM)：事务管理器，控制全局事务的边界，负责开启全局事务、提交或回滚全局事务。
    3. Resource Manager (RM)：资源管理器，控制分支事务，负责分支注册、状态汇报、驱动分支事务的提交或回滚。

    Seata的AT模式是如何工作的？

    1. 一阶段解析SQL：解析SQL语句，识别数据库操作类型（INSERT、UPDATE、DELETE），记录数据的前后镜像（Before Image和After Image）。
    2. 一阶段提交：生成回滚日志，将前后镜像和业务SQL组成回滚日志record，一并入库。
    3. 二阶段提交：如果全局事务成功，异步删除回滚日志记录。
    4. 二阶段回滚：如果全局事务失败，根据回滚日志记录生成反向SQL执行回滚。

    Seata的TCC模式与AT模式有什么区别？

    1. 侵入性：TCC模式需要业务代码实现Try、Confirm、Cancel三个方法，AT模式对业务代码无侵入。
    2. 实现复杂度：TCC模式需要业务自行设计补偿逻辑，实现复杂度高；AT模式由框架自动处理。
    3. 性能：TCC模式性能较高，因为直接操作资源；AT模式需要解析SQL和记录镜像。
    4. 适用场景：TCC适用于对性能要求高、对数据一致性要求严格的场景；AT适用于大多数业务场景。

    Seata的事务隔离是如何保证的？

    1. 全局锁：Seata通过全局锁保证事务间的隔离性。
    2. 写隔离：本地事务提交前，需要先获取全局锁。
    3. 读隔离：全局事务回滚时，通过undo log生成反向SQL进行补偿。
    4. 锁等待超时：获取全局锁超时时会自动回滚，保证系统不会无限等待。

    Seata如何与Spring Cloud Alibaba集成？

    1. 添加依赖：在项目中添加seata-spring-boot-starter依赖。
    2. 配置TC地址：在配置文件中指定Seata Server（TC）的地址。
    3. 配置数据源代理：将普通数据源配置为Seata的数据源代理。
    4. 使用@GlobalTransactional：在需要分布式事务的方法上添加@GlobalTransactional注解。

    Seata的高可用是如何实现的？

    1. TC集群部署：Transaction Coordinator支持集群部署，通过Nacos等注册中心实现服务发现。
    2. Session共享：集群节点间通过session共享实现状态同步。
    3. DB存储模式：使用数据库存储全局事务和分支事务信息。
    4. Redis存储模式：可选Redis存储模式，提高性能。

    Seata的死锁检测是如何工作的？

    1. 全局锁检测：Seata维护全局锁列表，检测锁冲突。
    2. 锁等待超时：获取锁超时后会自动回滚当前事务。
    3. 死锁避免：通过按序获取全局锁避免死锁发生。
    4. 锁释放：事务结束（提交或回滚）后立即释放全局锁。

    Seata的SAGA模式适用于什么场景？

    1. 长事务场景：适用于业务流程长、业务参与者多的场景。
    2. 跨系统集成：适用于需要集成多个已有系统服务的场景。
    3. 补偿型事务：需要业务自行定义补偿逻辑的场景。
    4. 状态机驱动：通过状态机定义业务流程，便于管理和监控。

    Seata的XA模式有什么特点？

    1. 强一致性：XA模式提供强一致性保证。
    2. 两阶段提交：基于标准的XA协议实现两阶段提交。
    3. 数据库支持：需要数据库支持XA协议。
    4. 性能开销：由于需要锁定资源，性能相对AT模式较低。

    Seata的事务传播机制是怎样的？

    1. 嵌套事务：支持在已开启全局事务的方法中嵌套新的全局事务。
    2. 事务传播行为：可以配置不同的传播行为，如REQUIRED、REQUIRES_NEW等。
    3. 独立事务：嵌套的全局事务可以设置为独立事务，不受外层事务影响。

    Seata如何处理分布式事务中的超时问题？

    1. 事务超时：可以配置全局事务的超时时间，超过时间未完成则自动回滚。
    2. 锁等待超时：获取全局锁的超时时间可以配置。
    3. 心跳检测：TC定期检测分支事务状态，超时未汇报则判定为失败。
    4. 定时回扫：TC有定时任务处理未完成的分支事务。
