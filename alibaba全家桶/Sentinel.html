    Sentinel

    Sentinel是什么？它主要用于解决什么问题？

    1. 流量控制：Sentinel是阿里巴巴开源的流量控制组件，主要用于处理分布式系统的流量控制。
    2. 熔断降级：当调用链路中某个服务出现故障或响应时间过长时，Sentinel可以自动进行熔断降级，防止故障蔓延。
    3. 系统负载保护：当系统的总体负载超过阈值时，Sentinel可以自动进行流量整形，保护系统不被压垮。
    4. 实时监控：提供实时监控和控制台，可以实时查看被保护服务的运行状态。
    Sentinel以流量为切入点，从流量控制、熔断降级、系统负载保护等多个维度帮助用户提升系统的稳定性和可用性。

    Sentinel与Hystrix的区别是什么？

    1. 社区活跃度：Sentinel由阿里巴巴开源，社区活跃度高，文档和示例丰富；Hystrix已进入维护模式。
    2. 功能完整性：Sentinel提供了更完整的流量控制、熔断降级和系统负载保护功能。
    3. 配置方式：Sentinel的配置更加灵活，支持通过控制台动态配置规则。
    4. 性能：Sentinel使用异步非阻塞模式，性能更优。
    5. 生态集成：Sentinel与Spring Cloud Alibaba有更好的集成。

    Sentinel中的流量控制（Flow Control）有哪些策略？

    1. 直接拒绝：当QPS超过阈值时，直接拒绝请求。
    2. 冷启动：允许系统预热一段时间，阈值从最小值逐渐增长到设定值。
    3. 匀速排队：让请求以匀速的方式通过，常用于处理突发的批量请求。
    4. 关联限流：当关联资源的QPS超过阈值时，限流当前资源。

    Sentinel的熔断降级是如何工作的？

    1. 慢调用比例：当资源的每秒请求数大于最小请求数，且慢调用的比例超过阈值时，触发熔断。
    2. 异常比例：当资源的每秒请求数大于最小请求数，且异常比例超过阈值时，触发熔断。
    3. 异常数：当资源的异常数超过阈值时，触发熔断。
    4. 恢复机制：熔断后会进入半开状态，如果后续请求成功，则会恢复正常。

    Sentinel中的@SentinelResource注解有哪些属性？

    1. value：资源名称，必需。
    2. blockHandler：处理BlockException的方法名。
    3. blockHandlerClass：blockHandler方法所在的类。
    4. fallback：处理熔断降级的方法名。
    5. fallbackClass：fallback方法所在的类。
    6. exceptionsToIgnore：忽略的异常类型，不进行降级处理。

    Sentinel的核心类有哪些？

    1. Entry：表示一次资源访问的入口，使用try-with-resources方式。
    2. Context：上下文对象，保存当前调用链的元数据。
    3. Node：秒级监控数据统计节点。
    4. ProcessorSlot：插槽接口，定义了各种资源检查和处理逻辑。

    Sentinel如何实现实时监控？

    1. 滑动窗口：Sentinel使用滑动窗口来统计秒级的请求数据。
    2. 异步推送：监控数据通过异步方式推送到控制台。
    3. LogBase：将监控数据写入本地文件。
    4. Client：通过HTTP请求将数据上报到控制台。

    Sentinel中的热点参数限流是什么？

    1. 参数索引：通过参数索引指定要限流的参数。
    2. 参数值限流：针对特定参数值进行限流。
    3. 配置规则：可以配置参数值对应的限流阈值。
    4. 使用场景：适用于对接口的某些参数进行精细化流量控制。

    Sentinel如何与Spring Cloud集成使用？

    1. 添加依赖：在项目中添加spring-cloud-starter-alibaba-sentinel依赖。
    2. 配置文件：配置Sentinel控制台地址等信息。
    3. 使用注解：在需要限流的方法上使用@SentinelResource注解。
    4. 控制台配置规则：通过Sentinel控制台动态配置限流规则。

    Sentinel中的系统规则（System Rule）有哪些类型？

    1. Load自适应：根据系统负载进行自适应保护，需要Linux系统。
    2. RT：所有资源的平均响应时间超过阈值时触发流量控制。
    3. 线程数：当单机器的线程数超过阈值时触发流量控制。
    4. QPS：当单机器的QPS超过阈值时触发流量控制。
    5. CPU使用率：当CPU使用率超过阈值时触发流量控制。

    Sentinel的滑动窗口算法是如何工作的？

    1. 窗口划分：将时间线划分为固定大小的窗口。
    2. 窗口滑动：窗口随时间推移不断向前滑动。
    3. 统计计算：聚合当前滑动窗口内的所有请求数据进行统计。
    4. 精确控制：可以精确到毫秒级别的流量控制。

    Sentinel如何实现服务熔断后的恢复？

    1. 半开状态：熔断器打开后，经过一定时间会进入半开状态。
    2. 试探请求：半开状态下会放行一个试探请求。
    3. 结果判断：如果试探请求成功，则关闭熔断器；否则继续打开。
    4. 恢复时间：可以配置熔断恢复的时间窗口。

    Sentinel控制台有哪些功能？

    1. 实时监控：查看资源的实时QPS、响应时间、通过量等。
    2. 规则配置：在线配置流控规则、熔断规则、系统规则等。
    3. 机器列表：查看已注册的应用节点信息。
    4. 动态规则：规则配置后会实时推送到应用节点生效。

    Sentinel中的网关限流有哪些类型？

    1. API维度：对API分组进行限流。
    2. 参数维度：对特定参数值进行限流。
    3. 自定义维度：通过自定义埋点实现更灵活的限流。
    4. 整合Gateway：提供与Spring Cloud Gateway的集成。
