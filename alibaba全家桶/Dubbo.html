    Dubbo

    Dubbo是什么？它有哪些核心功能？

    1. 高性能RPC通信：Dubbo是一款高性能的分布式服务框架，提供了RPC通信能力，支持多种通信协议。
    2. 服务治理：提供服务注册与发现、负载均衡、路由、容错等治理能力。
    3. 微服务集成：可以与Spring Cloud等微服务框架无缝集成。
    4. 多协议支持：支持Dubbo协议、REST、gRPC、Thrift等多种协议。
    Dubbo是阿里巴巴开源的分布式服务治理框架，目前是Apache顶级项目。

    Dubbo的工作原理是什么？

    1. 服务提供者（Provider）：启动时向注册中心注册自己的服务。
    2. 服务消费者（Consumer）：启动时向注册中心订阅自己需要的服务。
    3. 注册中心（Registry）：返回服务提供者地址列表给消费者，如有变更，注册中心会推送变更通知。
    4. 消费者调用提供者：消费者根据负载均衡策略选择一台提供者进行调用。
    5. 监控中心（Monitor）：统计服务调用次数、调用时间等数据。

    Dubbo支持哪些注册中心？

    1. Zookeeper：最常用的注册中心，基于CAP理论，保证一致性。
    2. Nacos：阿里巴巴开源的服务发现和配置管理平台。
    3. Consul：HashiCorp开源的服务发现和配置管理工具。
    4. Eureka：Netflix开源的服务注册中心，支持AP。

    Dubbo的负载均衡策略有哪些？

    1. Random LoadBalance：随机策略，按权重设置随机概率。
    2. RoundRobin LoadBalance：轮询策略，按公约后的权重设置轮询比例。
    3. LeastActive LoadBalance：最少活跃调用数策略，相同活跃数的随机。
    4. ConsistentHash LoadBalance：一致性Hash策略，相同参数的请求总是发到同一提供者。

    Dubbo的集群容错策略有哪些？

    1. Failover Cluster：失败自动切换，当出现失败时重试其他服务器。
    2. Failfast Cluster：快速失败，只发起一次调用，失败立即报错。
    3. Failsafe Cluster：失败安全，出现异常时，直接忽略。
    4. Failback Cluster：失败自动恢复，后台记录失败请求，定时重发。
    5. Forking Cluster：并行调用多个服务器，只要有一个成功即返回。
    6. Broadcast Cluster：广播调用所有提供者，逐个调用，任意一台报错则报错。

    Dubbo的通信协议有哪些？各有什么特点？

    1. Dubbo协议：默认使用Hessian序列化，TCP传输，适合小数据量大并发调用。
    2. RMI协议：Java标准远程调用协议，JDK自带，适合少并发场景。
    3. Hessian协议：基于HTTP的远程调用，使用Hessian序列化，适合传入传出参数较大时。
    4. HTTP协议：使用Spring的HttpInvoker，基于HTTP和JSON序列化。
    5. gRPC：Google开源的高性能RPC框架，支持双向流和流式处理。

    Dubbo的序列化方式有哪些？

    1. Hessian：Dubbo默认的序列化方式，性能较好。
    2. Dubbo序列化：阿里巴巴自己开发的高效序列化方式。
    3. JSON：使用Fastjson进行JSON序列化。
    4. Java原生序列化：JDK自带，效率较低。
    5. Avro：高性能序列化框架。

    Dubbo的Directory是什么？

    1. 服务目录：Directory存储了所有可用的服务提供者列表。
    2. RegistryDirectory：基于注册中心的服务目录，会动态更新。
    3. StaticDirectory：静态服务目录，服务列表固定不变。
    4. 作用：为负载均衡提供可调用的服务列表。

    Dubbo的Router是什么？

    1. 服务路由：决定服务消费者可调用哪些服务提供者的逻辑。
    2. 条件路由：基于条件表达式的路由规则。
    3. 脚本路由：基于脚本引擎（如JavaScript）的路由规则。
    4. 应用场景：用于服务隔离、灰度发布等场景。

    Dubbo的ConfigCenter配置中心有什么作用？

    1. 配置管理：集中管理Dubbo的配置。
    2. 动态配置：支持运行时动态修改配置，无需重启。
    3. 配置共享：实现多环境、多租户的配置共享。
    4. 支持多种后端：如Zookeeper、Nacos、Apollo等。

    Dubbo3的新特性有哪些？

    1. 应用级服务发现：从接口级发现升级到应用级发现，更适合云原生。
    2. Triple协议：基于HTTP/2的gRPC协议，兼容gRPC生态。
    3. 云原生支持：更好的Kubernetes支持。
    4. 性能提升：相比Dubbo2有20%以上的性能提升。

    Dubbo如何实现延迟暴露？

    1. delay配置：通过配置服务延迟暴露时间。
    2. 使用场景：用于服务依赖检查，等待依赖服务启动完成后再暴露。
    3. Spring集成：配合Spring的Bean初始化完成后暴露服务。

    Dubbo的服务降级实现方式有哪些？

    1. mock配置：通过配置服务降级处理类。
    2. 客户端降级：在消费端配置mock，调用失败时返回mock数据。
    3. 整合Sentinel：使用Sentinel实现更细粒度的降级和熔断。
    4. 返回null：最简单的降级方式，调用失败时返回null。
